---
title: "AgentPlane + Claude Code"
description: "Use Claude Code for edits while AgentPlane records task state, approval, verification, and finish inside Git."
---

Use this when Claude Code is changing files in a local repository and you want the work to remain
reviewable after the chat is gone.

## When to use

- You run Claude Code in the same checkout where Git history matters.
- You want a concrete task, plan, verification result, and finish record.
- You do not want a hosted workflow system between Claude Code and your repository.

## Copy-paste flow

```bash
npm i -g agentplane
agentplane init
agentplane quickstart

agentplane task new --title "Implement pricing copy update" --description "Update the pricing copy and verify the site build." --priority med --owner CODER --tag code
agentplane task plan set <task-id> --text "Update the copy, keep scope to the pricing page, run the site build." --updated-by CODER
agentplane task plan approve <task-id> --by ORCHESTRATOR
agentplane task start-ready <task-id> --author CODER --body "Start: pricing copy update with Claude Code; scope is limited to the approved page and build verification."
```

Then run Claude Code in the repository and ask it to follow the repo `AGENTS.md` policy.

After Claude Code edits files:

```bash
agentplane task verify-show <task-id>
# run the checks listed by verify-show
agentplane verify <task-id> --ok --by CODER --note "Approved checks passed after Claude Code edits."
agentplane finish <task-id> --author CODER --body "Verified: approved checks passed and the task scope is complete." --result "Pricing copy update completed" --commit <git-rev>
```

## Expected repo artifacts

- task README with the approved plan
- Git diff from Claude Code edits
- verification entry with the checks you ran
- finish record tied to the final commit

## Limitations

AgentPlane does not drive Claude Code or inspect Claude's private chat context. It records the
repository workflow around the code changes.
