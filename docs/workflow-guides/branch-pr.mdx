---
title: "branch_pr workflow"
description: "Use Agentplane branch_pr mode for per-task branches, PR artifacts, verification, and integration boundaries."
---

Use this when implementation should stay separate from the base checkout until review and hosted
checks are complete.

`branch_pr` is still the same harness split:

- The human chooses the task, review boundary, and integration policy.
- The agent creates the task branch or worktree, writes code, runs checks, and records evidence.
- Agentplane keeps PR artifacts, verification state, and finish evidence tied to the task.

## When to use

- Multiple people or agents may work near the same repository.
- Integration should happen only after review or hosted checks.
- You want PR metadata and verification artifacts per task.

## Handoff prompt

Give this to the coding agent instead of manually driving every command:

```text
Use Agentplane branch_pr mode. Start or resume the task in a dedicated worktree, keep the
implementation commit on the task branch, publish PR artifacts, record verification, and stop for
review before integration.
```

The implementation commit belongs to the task branch. The final close step records that commit from
the base checkout after integration.

## Expected repo artifacts

- task branch or dedicated worktree;
- `.agentplane/tasks/<task-id>/pr/` PR title/body/diffstat artifacts;
- task README verification evidence;
- base-branch finish record after integration;
- merged branch/worktree cleanup after hosted close.

Lifecycle/status commits, when a command supports them, are task-state checkpoints. They are not a
substitute for the implementation commit that finish records after integration.

## Operator reference

Use the [CLI reference](../reference/cli) for exact flags. The critical route commands are
`work start`, `pr open`, `verify`, `integrate`, `finish`, and `cleanup merged`.

For agents and maintainers checking lifecycle parity, the ordered route is:

```bash
agentplane work start <task-id> --agent <ROLE> --slug <slug> --worktree
agentplane task start-ready <task-id> --author <ROLE> --body "Start: ..."
git commit -m "Implement <task>"
agentplane task verify-show <task-id>
agentplane pr open <task-id> --branch task/<task-id>/<slug> --author <ROLE>
agentplane verify <task-id> --ok --by <ROLE> --note "Verified: ..."
bun run workflow:wait-remote-checks -- <pr-number>
agentplane integrate <task-id> --branch task/<task-id>/<slug> --run-verify
agentplane finish <task-id> --author INTEGRATOR --body "Verified: ..." --result "Merged" --commit <git-rev> --close-commit
```

## Limitations

`branch_pr` adds process overhead. Use `direct` for solo short loops where a separate branch or
worktree would slow down the task without improving review.
