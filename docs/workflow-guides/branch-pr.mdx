---
title: "AgentPlane + branch_pr workflow"
description: "Use AgentPlane branch_pr mode for per-task branches, PR artifacts, verification, and integration boundaries."
---

Use this when the implementation branch should stay separate from the base checkout until review and
verification are complete.

## When to use

- Multiple people or agents may work near the same repository.
- Integration should happen only after review or hosted checks.
- You want PR metadata and verification artifacts per task.

## Copy-paste flow

```bash
npm i -g agentplane
agentplane init
agentplane config set workflow_mode branch_pr

agentplane task new --title "Improve CLI help copy" --description "Update CLI help copy and verify docs generation." --priority med --owner CODER --tag code
agentplane task plan set <task-id> --text "Edit CLI help copy, regenerate docs if needed, run focused checks." --updated-by CODER
agentplane task plan approve <task-id> --by ORCHESTRATOR
agentplane work start <task-id> --agent CODER --slug cli-help-copy --worktree
```

In the task worktree:

```bash
agentplane task start-ready <task-id> --author CODER --body "Start: branch_pr CLI help copy update with focused verification."
# edit files
git add <changed-files>
git commit -m "Update CLI help copy"
agentplane task verify-show <task-id>
agentplane pr open <task-id> --branch task/<task-id>/cli-help-copy --author CODER
agentplane pr update <task-id>
agentplane verify <task-id> --ok --by CODER --note "Focused checks passed on the task branch."
```

Back on base checkout:

```bash
agentplane integrate <task-id> --branch task/<task-id>/cli-help-copy --run-verify
# optionally run remote-gate if repository policy requires it
agentplane finish <task-id> --author INTEGRATOR --body "Verified: branch integrated after review and verification." --result "CLI help copy updated" --commit <implementation-git-rev> --close-commit
agentplane cleanup merged --finalize
```

The `--commit` value is the already-created implementation commit from the task branch. Do not use
`finish --commit-from-comment` in `branch_pr`: finish runs from the base checkout, while
implementation commits belong to the task worktree.

## Expected repo artifacts

- task branch in the dedicated worktree
- `.agentplane/tasks/<task-id>/pr/` PR title/body/diffstat artifacts
- task README verification evidence
- base-branch finish record after integration
- merged branch/worktree cleanup via `agentplane cleanup merged --finalize`

Lifecycle/status commits, when a command supports them, are task-state checkpoints. They are not a
substitute for the implementation commit that `finish --commit <git-rev>` records after integration.

## Limitations

`branch_pr` adds process overhead. Use `direct` for solo short loops where a separate branch or
worktree would slow down the task without improving review.
