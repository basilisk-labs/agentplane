# Showcase

Agentplane grows best through visible evidence and concrete examples.

This page is for teams, maintainers, and public repositories that use Agentplane to make AI work
reviewable in Git.

## Submit an example

Open a GitHub Discussion with:

- Repository or project link.
- Agent used: agent-agnostic, local, editor, hosted, or custom runner.
- Workflow mode: `direct` or `branch_pr`.
- Evidence required before merge.
- Example task README, ACR, or PR link if public.

## Example formats

### First ACR

Share a task that produced:

```text
.agentplane/tasks/<task-id>/README.md
.agentplane/tasks/<task-id>/acr.json
```

### Agent PR policy

Share a contributor policy that requires AI-written PRs to include:

- task intent
- approved or waived plan
- verification evidence
- valid ACR
- finish record linked to the work commit

### Workflow guide

Share the exact flow your repo uses with a coding agent:

- agent type
- human approval boundary
- CI gate
- ACR gate
- `direct` or `branch_pr`

## Trust boundary

Do not submit secrets, private transcripts, raw prompts, credentials, or full terminal logs.
Commit-safe evidence is enough.
