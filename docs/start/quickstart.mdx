# Quickstart

This quickstart sets up the local harness. After setup, a coding agent should operate the task
lifecycle through the CLI while you set the task, guardrails, and review boundary.

## In 90 seconds you will

1. install Agentplane;
2. initialize a repository;
3. see the files Agentplane writes;
4. know what to hand to an agent;
5. know what to review before merging.

## Requirements

- Node.js 24+
- Git
- A local terminal

## Install Agentplane

```bash
npm i -g agentplane
```

If you do not want a global install, use `npx agentplane ...` for the same commands.

## Initialize a project

Run from a repository root:

```bash
agentplane init
```

This creates the repo-local policy and workflow surface that agents can inspect before mutating
files.

## Hand work to an agent

Give your coding agent the repository and the task. The agent should read the policy gateway,
operate the Agentplane lifecycle, run the relevant checks, and leave Git-visible evidence.

Use this prompt shape:

```text
Use Agentplane as the harness. Read AGENTS.md, create or continue the task, keep the diff inside the
approved scope, run the verification contract, and record the result before asking for review.
```

If you want to inspect the installed route yourself before handoff:

```bash
agentplane quickstart
```

## Review the result

After the agent finishes, review the repo evidence:

```text
.agentplane/tasks/<task-id>/README.md
.agentplane/tasks/<task-id>/acr.json
git diff
```

For trace-heavy workflows, the agent may also leave trace or export files:

```text
.agentplane/traces/<run-id>.json
.agentplane/exports/<run-id>.otel.json
```

## What changed in your repo?

Typical local files:

```text
AGENTS.md
.agentplane/WORKFLOW.md
.agentplane/tasks/<task-id>/README.md
.agentplane/tasks/<task-id>/acr.json
.agentplane/traces/<run-id>.json
.agentplane/exports/<run-id>.otel.json
```

Generated cache and derived context files can be rebuilt. Task records, traces, exports, and Agent
Change Records are the evidence trail.

## How do I remove Agentplane files?

Remove the local project artifacts you no longer want:

```bash
rm -rf .agentplane
rm -f AGENTS.md CLAUDE.md agentplane.yaml
```

If the repository already used `AGENTS.md` or `CLAUDE.md` for another purpose, review the file
before removing it.
