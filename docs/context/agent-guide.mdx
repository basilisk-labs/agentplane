# Agent guide to project context

Use this page when an agent needs project knowledge before editing code or docs.

## Before editing

```bash
agentplane context search "<task topic>"
agentplane context check
```

Read matching wiki pages and source refs before creating new terms, pages, facts, or graph edges.

For a generated context-assimilation task, first read its `context-pack.md`,
`source-set.lock.json`, `source-spans.skeleton.jsonl`, `canonical-snapshot.json`,
`canonical-entity-catalog.json`, `extraction-contract.json`, and `expected-artifacts.json`.

## When writing context

- Keep claims source-backed.
- Prefer updating canonical pages over creating duplicates.
- Record aliases when source wording differs from the canonical term.
- Keep ambiguous identities as conflicts or open questions.
- Use Markdown links for source refs and wiki links where they help later readers.
- Do not copy secrets or private credentials into context.

## Semantic entity reconciliation

Entity identity is an agent judgment, not a stable-ID or string-deduplication operation. For every
entity-bearing source term:

1. Inspect plausible candidates in the complete canonical entity catalog, then confirm them with
   context search, graph neighbors, Wiki use, and source refs.
2. Compare kind, scope, validity period, ownership, defining properties, aliases, provenance, and
   graph neighborhood.
3. Record `same_as`, `alias_of`, `distinct_entity`, `possibly_same_as`, or
   `new_entity_proposal` with candidates checked, comparison dimensions, evidence for and against,
   rationale, and unresolved questions.
4. Reuse `canonical_entity_id` for `same_as` and `alias_of`; do not create a second graph entity.
5. Keep both identities when decisive evidence is missing. Similar spelling is not proof.

Deterministic Agentplane code prepares the catalog, validates the decision record, and applies the
chosen identifier. It must not choose the semantic resolution itself.

## Page granularity

Create a separate page when the item is reusable:

- concept;
- entity;
- decision;
- requirement;
- risk;
- workflow;
- module;
- policy.

Keep smaller material under headings inside a broader page.

## Verification

Run the checks declared by the task. For context-bearing tasks, the usual minimum is:

```bash
agentplane context check
agentplane context search "<known new term>"
agentplane task verify-show <task-id>
```

If the task changed wiki pages, facts, graph rows, or provenance, record what changed and why in the
task verification note.

## Finish criteria

Context work is done only when:

- source set is named;
- new or changed claims have source refs;
- duplicate concepts were checked;
- generated projection was refreshed when required;
- reviewer can see coverage, conflicts, omissions, and follow-ups.
