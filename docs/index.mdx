# Agentplane documentation

Agentplane is a CLI-first operational workflow and observability layer for AI agents. It keeps
agent work inspectable through local workflows, traces, context, recipes, artifacts, and Git-native
evidence.

## Start in order

1. [Quickstart](start/quickstart) - install, init, run, and inspect the first trace.
2. [What Agentplane writes](start/files-written) - understand the repo-local artifacts.
3. [Agent Change Records](reference/acr) - understand evidence around AI-authored code.
4. [Workflow guides](workflow-guides) - use Agentplane with Claude Code, Codex, Cursor, or Aider.

## Most common jobs

- I want to try Agentplane locally -> [Quickstart](start/quickstart)
- I want to review AI-authored code -> [Agent Change Records](reference/acr)
- I want to inspect a run -> [Traces](concepts/traces)
- I want reusable agent behavior -> [Recipes](recipes)
- I want my agent to use repo knowledge -> [Context engineering](concepts/context-engineering)
- I want to compare Agentplane -> [Compare](compare)
