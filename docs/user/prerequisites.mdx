description: "What you need installed and what assumptions Agent Plane makes about your environment."
---

## Required

- Git repository access.
- Node.js 24 or newer.
- `npm` or `bun` available in PATH.

## Recommended

- `zsh` or `bash` shell.
- Working knowledge of `git status`, `git add`, and `git commit`.
- One IDE where your agents run.

## Runtime assumptions

- Work happens in a local repository checkout.
- Agent Plane commands execute on your machine.
- Outside-repo access and network access can require explicit approvals, based on config.

## Backend prerequisites

### Local backend

No external dependencies. Tasks live in `.agentplane/tasks`.

### Cloud backend

- Network access to your AgentPlane Cloud sync service.
- Required env keys:
  - `AGENTPLANE_CLOUD_ENDPOINT`
  - `AGENTPLANE_CLOUD_TOKEN`
  - `AGENTPLANE_CLOUD_PROJECT_ID`
- Optional env keys:
  - `AGENTPLANE_CLOUD_PROVIDER`

Provider-specific connectors, including Redmine, are configured in AgentPlane Cloud.

## Before you initialize

1. Open the target repository root in terminal.
2. Confirm node version with `node -v`.
3. Confirm git works with `git status`.
