## Help and version

```bash
agentplane --help
agentplane --version
```

## Quickstart

```bash
agentplane quickstart
```

## Init

```bash
agentplane init
```

If the directory is not a git repository, `agentplane init` will initialize git, pin the base branch,
and create an install commit with the current Agentplane version. In existing repos, it pins the base
branch to the current branch and commits the install artifacts it creates.

Non-interactive:

```bash
agentplane init --yes --require-plan-approval yes --require-network-approval no
```

## Config

```bash
agentplane config show
agentplane config set workflow_mode branch_pr
```

## Tasks

```bash
agentplane task list
agentplane task show <task-id>
agentplane task new --title "..." --description "..." --priority med --owner CODER --tag nodejs
agentplane task doc set <task-id> --section Summary --text "..."
```

## Workflow

```bash
agentplane start <task-id> --author CODER --body "Start: ..."
agentplane verify <task-id>
agentplane finish <task-id> --author INTEGRATOR --body "Verified: ..."
```

## Branching (branch_pr)

```bash
agentplane work start <task-id> --agent CODER --slug <slug> [--worktree]
agentplane pr open <task-id>
agentplane pr update <task-id>
agentplane pr check <task-id>
agentplane integrate <task-id> --branch task/<task-id>/<slug> --run-verify
```

```bash
agentplane branch base set --current
agentplane branch base clear
agentplane branch base explain
```

## Recipes

```bash
agentplane recipes list
agentplane recipes explain <id>
agentplane scenario list
agentplane scenario run <recipe:scenario>
```
