## Overview

Agent Plane reads `.agentplane/WORKFLOW.md` and validates the YAML front matter as the project source of truth.
The CLI still accepts legacy `.agentplane/config.json` as an import fallback for older workspaces, but saves
changes back to `WORKFLOW.md`.

Use CLI commands to read and update values safely:

```bash
agentplane config show
agentplane config set workflow_mode branch_pr
```

## High-impact keys

### workflow.mode

- `direct`: single checkout workflow.
- `branch_pr`: per-task branches and PR artifacts.

### status_commit_policy

- `off`: do not gate status/comment-driven commits.
- `warn`: allow, but warn when no explicit confirmation.
- `confirm`: require explicit confirmation.

### finish_auto_status_commit

- `false` by default.
- `init` sets it to `true` for `branch_pr`, `false` for `direct`.

### approvals

- `require_plan`
- `require_network`
- `require_verify`
- `require_force`

These keys control approval gates; they do not add or remove CLI capabilities.

### workspace

- `workspace.agents_dir`
- `workspace.workflow_dir`
- `workspace.worktrees_dir`
- `workspace.tasks_path` (export-only snapshot path)

### tasks.tags (primary tag model)

- `tasks.tags.primary_allowlist`
- `tasks.tags.strict_primary`
- `tasks.tags.fallback_primary`
- `tasks.tags.lock_primary_on_update`

Task policy resolves exactly one primary tag per task. Secondary tags remain free-form.

### tasks.verify

- `tasks.verify.require_steps_for_primary`
- `tasks.verify.require_verification_for_primary`
- `tasks.verify.spike_tag`
- `tasks.verify.enforce_on_plan_approve`
- `tasks.verify.enforce_on_start_when_no_plan`

Compatibility note: `tasks.verify.required_tags` still exists in schema and may be present in older configs.
Current enforcement should be documented and configured through primary-based keys.

### tasks.backend

- `tasks.backend.config_path` points to the active backend config file.

### execution

- `execution.profile`
- `execution.reasoning_effort`
- `execution.text_verbosity`
- `execution.tool_budget`
- `execution.stop_conditions`
- `execution.handoff_conditions`
- `execution.unsafe_actions_requiring_explicit_user_ok`

`execution.reasoning_effort` controls depth, not answer length. `medium` is the balanced default;
`low` is latency-oriented; `high` and `xhigh` should be reserved for conservative profiles or
eval-backed work. `execution.text_verbosity` controls output length/structure in runner prompt
contracts and is rendered into the execution profile runtime block.

### runner

- `runner.default_adapter`
- `runner.trace.*`
- `runner.timeouts.*`

`runner.default_adapter` selects the shared runner adapter (`codex` or `custom`).

### runner.custom

- `runner.custom.command`
- `runner.custom.env`
- `runner.custom.enforcement.mode`
- `runner.custom.enforcement.platform`

Current supported enforcement modes:

- `none` (default): the custom adapter forwards recipe/runtime intent through env only. Sandbox is
  advisory, not enforced.
- `codex_sandbox_full_auto`: the custom adapter wraps the configured command in
  `codex sandbox <platform> --full-auto` when the recipe requests a supported sandbox.

Current supported wrapper-enforced sandbox values:

- `workspace-write`

`read-only` is intentionally unsupported for the `custom` wrapper path. The reason is not just
"not implemented": the shared runner contract requires the wrapped process to write `result.json`
and trace artifacts under `run_dir`, while the default `codex sandbox <platform>` mode blocks
writes to both `cwd` and `TMPDIR`. Until the wrapper path can express a narrower writable allowlist
for those required artifacts, the contract must stay fail-closed.

If wrapper mode is configured, inspect `agentplane task run <task-id> --dry-run` or
`agentplane task run show <task-id>` for the `capability[sandbox]` and `policy_field[sandbox]`
lines. They surface the configured wrapper mode, capability level, and supported sandbox values
without requiring a manual read of `bundle.json`.

## Examples

Set primary tags that require Verify Steps:

```bash
agentplane config set tasks.verify.require_steps_for_primary '["code","data","ops"]'
```

Set primary tags that require recorded verification before finish:

```bash
agentplane config set tasks.verify.require_verification_for_primary '["code","data","ops"]'
```

Switch to branch workflow:

```bash
agentplane config set workflow_mode branch_pr
```

Enable stricter force approvals:

```bash
agentplane config set agents.approvals.require_force true
```

Configure the custom runner to use the Codex full-auto sandbox wrapper:

```bash
agentplane config set runner.default_adapter custom
agentplane config set runner.custom.command '["node","./scripts/custom-runner.mjs"]'
agentplane config set runner.custom.enforcement.mode codex_sandbox_full_auto
agentplane config set runner.custom.enforcement.platform auto
```
