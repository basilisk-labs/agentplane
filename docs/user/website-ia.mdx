# Website information architecture

The public site should make AgentPlane legible before it explains internal mechanics.

## Positioning stack

- Tagline: Git-native infrastructure for traceable AI work.
- One-liner: AgentPlane turns AI-agent changes into reviewable Git evidence: task, plan,
  verification, ACR, and closure.
- Category: repo-local evidence layer for AI-authored software changes.
- Wedge: every serious agent change should leave an Agent Change Record.
- Promise: use any coding agent; keep the evidence in Git.
- Trust: repo-local evidence, Git-visible review trail, no vendor lock-in.

## Top-level navigation

- Home
- Why
- ACR
- Workflows
- Docs
- Compare
- GitHub

Recipes stay below the core flow until the extension catalog is large enough to justify top-level
navigation.

task lifecycle, workflow modes, recipes, and backends. Framework internals stay in the direct-link
maintainer docs and are not part of the public user path.
