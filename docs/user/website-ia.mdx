# Website information architecture

The public site should make Agentplane legible before it explains internal mechanics.

## Positioning stack

- Tagline: Git-native infrastructure for traceable AI work.
- One-liner: Agentplane turns AI-agent changes into reviewable Git evidence: task, plan,
  verification, ACR, and closure.
- Category: repo-local evidence layer for AI-authored software changes.
- Wedge: every serious agent change should leave an Agent Change Record.
- Promise: install and initialize Agentplane, then let any coding agent operate inside the repo
  workflow.
- Trust: agent-agnostic repo-local evidence, Git-visible review trail, no vendor lock-in.

Agentplane is agent-agnostic because it works through files, Git, and the universal `AGENTS.md` /
`CLAUDE.md` policy gateway instead of depending on one model, IDE, or hosted agent platform.

The first-time path should show the default operating model before advanced command details:

```text
install -> init -> context init -> hand off to an agent -> review task evidence
```

Manual CLI usage beyond setup and recovery is an advanced operator path. Framework internals stay in
the direct-link maintainer docs and are not part of the public user path.
