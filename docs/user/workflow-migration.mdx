title: "Workflow contract recovery"
description: "How to repair .agentplane/WORKFLOW.md and its last-known-good snapshot."
---

## Target state

Each repository should have:

- `.agentplane/WORKFLOW.md` validated by schema + strict template checks.
- `.agentplane/workflows/last-known-good.md` snapshot.
- `agentplane doctor` passing without `ERROR`.

## Repair steps

1. Validate the candidate contract:

```bash
agentplane workflow build --validate --dry-run
```

2. Apply safe doctor fixes:

```bash
agentplane doctor --fix
agentplane doctor
```

3. Rebuild the active workflow contract when needed:

```bash
agentplane workflow build --validate
```

4. If the active workflow was corrupted, restore from the snapshot:

```bash
agentplane workflow restore
agentplane doctor
```

## Kill-switch (emergency only)

Set environment variable to disable workflow contract enforcement temporarily:

```bash
export AGENTPLANE_WORKFLOW_ENFORCEMENT=off
```

Effects:

- `agentplane preflight` skips workflow validation.
- `agentplane doctor` skips workflow contract checks.

Recovery:

1. Remove the variable.
2. Run `agentplane workflow build --validate --dry-run`.
3. Run `agentplane doctor`.
