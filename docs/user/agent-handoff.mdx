## Default usage model

AgentPlane is designed for agent-operated repository work.

Humans install and initialize AgentPlane. Coding agents operate the task lifecycle. Reviewers inspect
the evidence left in Git.

```text
install -> init -> context init -> hand off to an agent -> review task evidence
```

Manual CLI operation after initialization is an advanced path for operators, maintainers, recovery,
and debugging. The normal product path is to let the coding agent read the repository policy gateway
and run the AgentPlane lifecycle around its own work.

## Agent-agnostic by design

AgentPlane does not depend on one coding agent, model, IDE, or hosted agent platform.

It works at the repository boundary:

- `AGENTS.md` or `CLAUDE.md` gives agents a universal policy gateway.
- `.agentplane/WORKFLOW.md` records the workflow and configuration contract.
- `.agentplane/tasks/` stores task state, plans, verification, PR artifacts, and ACR files.
- `context/**` stores optional source-backed local context for agents.
- Git stores the final diff, branches, commits, and review trail.

Any agent that can read files, run shell commands, edit the repository, and respect `AGENTS.md` can
work inside the AgentPlane contract.

## What to give the agent

After setup, give your coding agent the repository and this instruction:

```text
Use AgentPlane for this repository.

Start with:
agentplane quickstart
agentplane role ORCHESTRATOR

Follow AGENTS.md and the workflow mode in .agentplane/WORKFLOW.md.
Create or reuse an AgentPlane task before mutating repository files.
Record the plan, start-ready state, verification evidence, and finish state through AgentPlane.
If the task needs reusable project knowledge, use the local context docs and context commands.
```

Use `ap` instead of `agentplane` when the repository policy says `ap` is available and preferred.

## First successful agent task

The first success criterion is not that a human learned every CLI command. It is that the agent
leaves a reviewable repository trail:

```text
.agentplane/tasks/<task-id>/README.md
.agentplane/tasks/<task-id>/acr.json
```

In `branch_pr` mode, the agent may also create:

```text
.agentplane/tasks/<task-id>/pr/**/*
```

## When humans should use the CLI directly

Use manual CLI docs when you are doing one of these:

- Initial setup or context initialization.
- Recovering a broken workflow or partial upgrade.
- Inspecting configuration, task state, or generated evidence.
- Operating backend sync.
- Maintaining AgentPlane itself.

For day-to-day feature work, bug fixes, documentation updates, and refactors, hand the repository to
the coding agent and review the evidence it records.
