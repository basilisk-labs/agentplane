## Overview

The local backend stores tasks in the repository under `.agentplane/tasks/` and is the default backend after init.

## Storage layout

- Task data: `.agentplane/tasks/<task-id>/README.md`
- Snapshot export (export-only, gitignored by default): `.agentplane/tasks.json`

The README file is the single source of truth for a task and includes metadata, comments, append-only events, and doc sections.

## Backend config

Local backend config is stored under `.agentplane/backends/local/backend.json` and referenced by `.agentplane/config.json` in `tasks_backend.config_path`.

Example:

```json
{
  "id": "local",
  "version": 1,
  "settings": {
    "dir": ".agentplane/tasks"
  }
}
```

## Notes

- Local backend supports full task CRUD and doc updates.
- Use `agentplane task export` to generate `.agentplane/tasks.json` for integrations.
