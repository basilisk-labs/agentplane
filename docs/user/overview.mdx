## What is Agentplane?

Agentplane is an agent-agnostic harness for governed coding-agent work, built for reproducible
runs, inspectable traces, local context, recipes, artifacts, and Git-native evidence.

It does not replace the model or agent framework. It gives agent work an operating surface:
workflows, context boundaries, tool-call records, verification checks, run artifacts, traces, and
Agent Change Records.

The operating split is deliberate:

- The agent writes code, uses the CLI, follows `AGENTS.md`, and records verification.
- The human sets tasks, adjusts guardrails, approves risky boundaries, and reviews evidence.
- Agentplane keeps that work durable as repo-local state instead of private chat or IDE context.

Use your existing coding agent to edit code. Use Agentplane to make the agent's work reviewable
after the chat window, terminal session, or IDE context is gone.

```text
task -> plan -> approve -> implement -> verify -> finish
```

The default path is:

```text
install -> init -> hand off to an agent -> review task evidence
```

Run `context init` only when the repository needs durable local knowledge. It is independent from
the prompt assembly path that gives agents policy, owner profile, route, recipe, skill, and task
context.

Manual CLI usage beyond setup, recovery, and review is an advanced operator path. User guides should
teach the harness boundary first; the command reference holds the exhaustive flags.

## Agent-agnostic model

Agentplane does not depend on one coding agent, model, IDE, or hosted platform. It aims for an
agent-agnostic and LLM-agnostic result: the same task boundary, policy gateway, verification
evidence, and Git review trail should work across tools.

It works at the repository boundary:

- `AGENTS.md` or `CLAUDE.md` gives agents a universal policy gateway.
- `.agentplane/WORKFLOW.md` records workflow and configuration.
- `.agentplane/tasks/` records plans, verification, PR artifacts, finish state, and ACR.
- `context/**` optionally records source-backed local context for agents.
- Git records diffs, branches, commits, and review history.

Any coding agent that can read files, run shell commands, edit the repository, and respect the
policy gateway can operate inside Agentplane.

## What Agentplane does not do

- It does not generate code for you.
- It does not choose a model.
- It does not replace Git.
- It does not replace CI.
- It does not replace pull request review.

## What does Agentplane give you?

- Repo-local task records.
- Policy gateway through `AGENTS.md` or `CLAUDE.md`.
- Explicit task lifecycle.
- Local workflow runs.
- Trace records.
- Verification evidence.
- Local context management: raw sources, wiki pages, facts, graph edges, projection, search, and
  task-context verification.
- ACR generation and validation.
- `direct` and `branch_pr` workflows.

## First artifact before first abstraction

The fastest way to understand Agentplane is to install it, initialize the repository, then hand the
approved task to a coding agent.

```bash
npm i -g agentplane
agentplane init
```

The visible payoff is a repo-owned task trail. The agent can create and update it through the CLI;
the human reviews it in Git:

```text
.agentplane/tasks/<task-id>/README.md
.agentplane/tasks/<task-id>/acr.json
```

For research-heavy tasks, local context adds a second repo-owned trail:

```text
.agentplane/cache.sqlite
.agentplane/context/derived/**/*
context/wiki/**/*
context/facts/**/*.jsonl
context/graph/**/*.jsonl
```

This is Agentplane's implementation of the LLM Wiki pattern for repositories: agents maintain a
source-backed knowledge layer in Git instead of re-reading every raw source on every question.

## Source of truth model

- Workflow policy: `AGENTS.md` or `CLAUDE.md`.
- Repo workflow/configuration: `.agentplane/WORKFLOW.md`.
- Task state: `.agentplane/tasks/`.
- Local context source artifacts: `context/**`.
- Local context derived artifacts: `.agentplane/context/derived/**`.
- Shared local projection cache: `.agentplane/cache.sqlite`.
- ACR evidence projection: `.agentplane/tasks/<task-id>/acr.json`.
- Command behavior: installed `agentplane` CLI.
- Git history: commits, branches, and review workflow.

`context/**` is the editable source layer. `.agentplane/context/derived/**` and
`.agentplane/cache.sqlite` are generated/disposable and can be rebuilt with
`agentplane context reindex`.
