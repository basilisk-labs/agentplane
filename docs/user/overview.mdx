## What AgentPlane is

AgentPlane is a local CLI for agent-driven development inside a Git repository.

It makes Claude Code, Codex, Cursor, Aider, and similar coding-agent work auditable by recording the
workflow around the code:

```text
task -> plan -> approve -> implement -> verify -> finish
```

AgentPlane runs locally. It does not add a hosted runtime between your repository and your agent.

## What it gives you

- A repo policy gateway: `AGENTS.md` or `CLAUDE.md`.
- A repo-local workspace: `.agentplane/`.
- Task records with ownership, plan, status, and dependencies.
- Explicit verification state.
- Finish records tied to Git revisions.
- Workflow modes for fast local loops and stricter branch/PR handoff.

## What it does not do

- It is not an agent.
- It is not a hosted agent platform.
- It is not a prompt framework.
- It does not replace Git.
- It does not replace your editor or terminal.
- It does not replace Claude Code, Codex, Cursor, Aider, or other coding agents.

## How the pieces fit

1. `agentplane init` creates the policy gateway and `.agentplane/` workspace.
2. A task records what the agent is supposed to solve.
3. A plan records the intended approach before implementation.
4. Verification records what was checked.
5. Finish records why the work is considered closed.

## Source of truth model

- Workflow policy: `AGENTS.md` or `CLAUDE.md`.
- Repo configuration: `.agentplane/config.json`.
- Task state: `.agentplane/tasks/`.
- Command behavior: installed `agentplane` CLI.
- Git history: commits, branches, and review workflow.
