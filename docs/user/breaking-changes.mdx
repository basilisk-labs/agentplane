## Summary

Current AgentPlane repositories are driven by the Node CLI `agentplane` with repository state under
`.agentplane/`. This page records active compatibility boundaries that matter for current upgrades.

## Current compatibility boundaries

1. Repository state lives under `.agentplane/`; root-level legacy workflow files are not read as
   fallbacks.
2. Initialization and framework upgrades run through `agentplane init` and `agentplane upgrade`.
3. New task READMEs use `doc_version=3`; use `agentplane task migrate-doc --all` only when
   `agentplane doctor` reports active v2/v3 migration drift after an upgrade.
4. `branch_pr` integrates task branches with merge commits by default so branch history is
   preserved. Use `--merge-strategy squash` only when compacting history is intentional.
5. `direct` remains supported, but protected-base repositories should use hosted PR merge and
   hosted close automation for `branch_pr` work.
6. Redmine backend configuration is env-first for credentials and field IDs.

## Upgrade checklist

1. Install or update the CLI and run `agentplane upgrade`.
2. Run `agentplane doctor`.
3. If doctor reports active task README migration drift, run `agentplane task migrate-doc --all`.
4. Regenerate generated docs only when command specs changed.

## Notes

- `AGENTS.md` is the canonical policy file for agent behavior in a repo; treat it as source-controlled infrastructure.
- The tasks JSON export is an advanced generated snapshot; inspect the current command contract with
  `agentplane help --all` and do not edit the snapshot by hand.
