title: "Code Map recipe"
description: "Install the code-map recipe to add a lightweight code-map discipline to agent development work."
---

`code-map@0.1.0` is the first production recipe in the signed Agentplane recipe catalog.

It adds a lightweight discipline for non-trivial coding tasks: inspect or refresh a compact project
code map before implementation, then report whether the map informed the change.

## What it changes

The recipe contributes:

- a recipe-local agent profile: `code-map-operator`;
- a skill: `code-map-discipline`;
- a prompt module that asks agents to check the code map before broad code changes;
- a scenario asset named `code-map`;
- an expected project artifact path, written as docs/code-map.md in the target repository.

The effect should be visible in agent behavior: for multi-module tasks, the agent should identify
entry points, ownership boundaries, and verification commands before editing.

## When to use it

Use `code-map` when:

- a coding task spans multiple modules or ownership boundaries;
- a new agent needs a compact map of the repository's runtime surfaces;
- a change depends on understanding entry points and verification commands.

Avoid it when:

- the task is a one-line change with an obvious owner;
- the repository already has a current code map for the touched area.

## Install

```bash
agentplane recipes list-remote --refresh --yes
agentplane recipes install code-map --refresh --yes
agentplane recipes list
agentplane recipes info code-map
```

## Add to a project

```bash
agentplane recipes add code-map
agentplane recipes active
agentplane recipes explain code-map
```

After `recipes add`, the recipe package is copied into:

```text
.agentplane/recipes/packages/code-map/
```

Agentplane records the active recipe in:

```text
.agentplane/recipes/registry.json
```

## Initialize with the recipe

If the recipe is available to the init flow, initialize a project with it enabled:

```bash
agentplane init --recipes code-map
```

## Expected project artifact

The recipe's scenario declares this output:

```text
docs/code-map.md
```

Keep that file compact. It should map the current task-relevant code surfaces, not become a full
architecture document.

## Limits

`code-map` does not run a separate hosted service and does not replace the normal task lifecycle. It
changes project-local recipe assets and prompt behavior; task creation, verification, and finish
still go through the standard Agentplane commands.
