## Top-level directories

- `packages/agentplane/`: published `agentplane` CLI package
- `packages/core/`: published `@agentplaneorg/core` package
- `packages/spec/`: canonical schemas and examples
- `packages/recipes/`: recipe runtime helpers
- `packages/testkit/`: shared test helpers
- `docs/`: user and developer documentation
- `scripts/`: repository maintenance and sync scripts
- `.github/workflows/`: CI and publish pipelines

## CLI package layout

- `packages/agentplane/src/cli/`: parser, help rendering, command wiring
- `packages/agentplane/src/commands/`: command handlers and domain logic
- `packages/agentplane/src/backends/`: task backend adapters
- `packages/agentplane/assets/`: framework templates copied by `init` and used by `upgrade`
- `packages/agentplane/dist/`: built output used by runtime and publish

## Core and spec layout

- `packages/spec/schemas/*.schema.json`: canonical schema source
- `packages/core/schemas/*.schema.json`: synchronized mirror for runtime package
- `scripts/sync-schemas.mjs`: sync/check tool for schema parity
- `scripts/sync-agent-templates.mjs`: sync/check tool for agent profile parity (`assets/agents` vs `.agentplane/agents` in this repository)

## Project runtime directories

After `agentplane init`, a project uses:

- `.agentplane/config.json`: runtime config
- `.agentplane/agents/*.json`: local agent profiles
- `.agentplane/tasks/<task-id>/README.md`: canonical task documents
- `.agentplane/tasks.json`: export snapshot for external integrations
- `.agentplane/cache/`: runtime cache (index/update-check metadata)
- `.agentplane/.upgrade/`: temporary/diagnostic upgrade artifacts
- `.agentplane/.release/`: temporary release planning artifacts

## Build and release artifacts

- `packages/**/dist/`: compiled package output
- `docs/releases/v*.md`: release notes consumed by release commands
- `agentplane-upgrade.tar.gz`: upgrade bundle generated in publish workflow

## Quick orientation

If you are changing command behavior:

1. Start from `packages/agentplane/src/cli/run-cli.ts`.
2. Follow command entry in `packages/agentplane/src/cli/run-cli/command-catalog.ts`.
3. Edit handler in `packages/agentplane/src/commands/**`.
4. Update docs and command tests in the same task.
