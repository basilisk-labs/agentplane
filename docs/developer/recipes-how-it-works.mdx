This page describes runtime behavior behind `recipes` and `scenario` commands.

## Install flow

1. Resolve source (`--name`, `--path`, or `--url`).
2. Download when source is remote.
3. Extract archive and locate `manifest.json`.
4. Validate manifest, recipe-local file references, and scenario definitions.
5. Copy the self-contained recipe bundle into `.agentplane/recipes/<recipe-id>/`.
6. Write project-local install metadata and preserve existing `runs/` artifacts.
7. Keep recipe-local `agents/`, `skills/`, `tools/`, and `scenarios/` inside the recipe directory.

## Listing and inspection

- `recipes list` scans project-local installed recipes and prints short entries.
- `recipes info` and `recipes explain` read manifest-backed details.
- `scenario list` enumerates resolver-backed scenario descriptors from installed recipes.
- `scenario info` prints manifest-backed scenario details and normalized `run_profile` data for one `<recipe:scenario>`.

## Scenario run flow

`scenario run`:

1. Parses `<recipe:scenario>` id.
2. Resolves the scenario through the deterministic recipe resolver.
3. Validates compatibility, recipe-local files, and the internal `scenario.json` contract.
4. Prints a prepared run plan with selection reasons and normalized `run_profile`.
5. Stops without executing tools or creating `runs/` artifacts.

## Environment behavior

Install and validation flows inherit the current process environment.
Recipe-local tool execution env is intentionally out of scope until a real orchestration runner lands.

## Caching behavior

- Remote index cache is stored globally, but installed recipes are project-local.
- `recipes list-remote --refresh` forces index refresh.
- `recipes cache prune` removes stale cached entries and supports `--dry-run`.

## Failure behavior

Common hard failures:

- missing or invalid manifest
- unsupported archive structure
- unknown scenario id
- missing recipe-local files referenced by manifest
- incompatible runtime context for the selected scenario
- network/download errors for remote sources

On failure, CLI exits non-zero and reports an error with standard error code mapping.
