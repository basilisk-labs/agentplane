# Release Notes — v0.3.16

## Summary

- Completed the refactor-roadmap candidate that moved the project into the release phase, including the package build-order fixes required for `core`, `agentplane`, and `testkit` to build together reliably.
- Strengthened the release and prepublish pipeline after `v0.3.15` by splitting fast and heavy checks, making release candidate routing fail earlier, and recording hosted publish evidence in task closure.
- Stabilized branch-pr workflow operations around hosted closure, remote branch reuse, and redundant close-tail behavior so release candidates can move through review and merge with less manual recovery.
- Added CI and test hardening for release-adjacent flows, including Node runtime upgrades, runtime-source isolation, deterministic branch-pr release tests, and platform-critical package build checks.

## Added

- Added the completed refactor-roadmap release candidate from PR #491, bundling the roadmap implementation, release-readiness fixes, and patch publication preparation into the `v0.3.16` baseline.
- Added explicit release prepublish phases so the fast gate can fail before slower heavyweight checks run, improving failure locality for release operators.
- Added hosted publish evidence recording to release task closure, making release completion auditable from the task history instead of only from external workflow state.
- Added deterministic release-task closure records for the `J57Q9H`, `G7YHZY`, `NT02DV`, `WGK57W`, `507P6G`, `MA7MQ4`, `N6XPEJ`, and `JWBEB7` work items that fed this patch.

## Improved

- Improved release build reliability by breaking the `agentplane` and `@agentplane/testkit` build cycle: publishable `agentplane` builds no longer compile repo-only testing compatibility files, and CI now builds `core`, then `agentplane`, then `testkit`.
- Improved release candidate safety with an early preflight for branch-pr release routes, preventing mutation when the checkout is not on the correct candidate path.
- Improved branch-pr closure behavior by avoiding redundant manual close tails after hosted closure has already reconciled the task state.
- Improved PR opening behavior when the remote branch already exists, reducing manual cleanup when release or workflow retries reuse an existing branch name.
- Improved CI compatibility by moving release workflows off deprecated Node 20 action usage where the project controls the runtime selection.
- Improved release-test determinism by pinning branch-pr release tests to explicit base and candidate branches instead of relying on the host machine's default `git init` branch.
- Improved runtime-source tests by isolating them from ambient `AGENTPLANE_*` environment variables, reducing false failures in developer shells and CI jobs.
- Improved package installation and framework bootstrap documentation so the manual recovery path consistently builds `core`, `agentplane`, and `testkit` in the order expected by CI.

## Fixed

- Fixed the package build-order regression that caused `@agentplane/testkit` and `agentplane` to require each other's `dist` output during prepublish checks.
- Fixed release CI failures caused by stale generated bootstrap documentation that referenced the old runtime-source output path.
- Fixed release apply test instability on GitHub Actions, where the temporary git repository defaulted to a non-`main` branch and made a base-branch assertion report the candidate-branch diagnostic.
- Fixed release task closure drift by recording the merged release, workflow, tests, and CI close commits that had already landed through the hosted route.

## Upgrade Notes

- No manual migration is required when upgrading from `v0.3.15` to `v0.3.16`.
- Developers working from a fresh framework checkout should run `bun run framework:dev:bootstrap`; the documented manual fallback now builds `@agentplaneorg/core`, `agentplane`, and `@agentplane/testkit` in that order.
