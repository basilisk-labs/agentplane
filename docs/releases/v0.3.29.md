# Release Notes - v0.3.29

## Summary

- Makes the `branch_pr` lifecycle safer by resolving lifecycle context explicitly, preserving branch history during integration by default, typing PR artifact state, and making PR creation transactional.
- Tightens task closure and runner behavior by routing no-op close transitions through one command path and verifying run repository contracts.
- Reduces CLI and tooling ambiguity by normalizing resume context output, sharing generated-text freshness helpers, and adding reusable branch PR fixture builders for tests.
- Updates public documentation so the documented workflow, examples, and runtime behavior match the current branch PR implementation.

## Added

- Added a branch PR lifecycle context resolver so task commands can derive the active branch, worktree, and artifact context through one explicit runtime path.
- Added transactional PR opening for branch PR tasks so local artifact sync, branch publication, and hosted PR creation are treated as one coordinated operation.
- Added typed PR artifact state to reduce ad hoc JSON handling around PR metadata, verification logs, review notes, and branch bookkeeping.
- Added a shared branch PR artifact fixture builder for tests, replacing repeated local fixture setup with a single reusable testkit helper.
- Added generated-text freshness helpers so tooling checks can share the same comparison and rewrite behavior instead of duplicating file-update logic.
- Added release coverage for the v0.3.28 evidence commit so the patch history includes the previous publication closeout in the next release delta.

## Improved

- Improved branch PR integration by preserving branch history by default instead of squashing task branches unless squash is requested explicitly.
- Improved the integration command contract so `--merge-strategy squash` is an intentional override rather than the default behavior.
- Improved public branch PR documentation to describe the current lifecycle, merge behavior, hosted close path, and task artifact expectations.
- Improved the workflow policy text so agent guidance now matches the runtime default merge strategy for branch PR integration.
- Improved PR opening reliability by making task artifact refresh part of the committed task branch state before hosted PR publication.
- Improved lifecycle mutation behavior by routing no-op closure transitions through the same transition command path used by real state changes.
- Improved closure consistency by refreshing task artifacts after lifecycle transition changes were committed.
- Improved runner repository contract coverage so run behavior is checked against explicit repository expectations instead of implicit assumptions.
- Improved task artifact maintenance around runner changes by refreshing the task metadata after the runner contract update landed.
- Improved CLI resume-context output by normalizing direct stdio surfaces and reducing context formatting drift.
- Improved CLI task artifact accuracy by refreshing task metadata after the stdio normalization commit.
- Improved branch PR fixture coverage by refreshing task artifacts after extracting the reusable test fixture builder.
- Improved generated-text tooling by refreshing task artifacts after the freshness helper extraction.
- Improved release readiness by closing each completed branch PR task with hosted merge evidence and deterministic closeout commits.
- Improved reviewability of the recent lifecycle work by keeping implementation commits and close commits visible in main history.
- Improved task traceability by retaining separate commits for the branch PR lifecycle resolver, PR artifact typing, transactional PR opening, merge behavior, docs alignment, transition service, runner contracts, stdio cleanup, testkit fixtures, and tooling helpers.
- Improved branch PR testing ergonomics by reducing duplicated setup across tests that need local PR artifact files.
- Improved maintenance of generated artifacts by centralizing freshness checking in shared helpers.
- Improved agent-facing workflow guidance by aligning `AGENTS.md` and policy modules with the current code path.
- Improved release auditability by generating this release from the recorded `.agentplane/.release/plan` delta since `v0.3.28`.
- Improved protected-main compatibility by keeping branch PR changes on task branches and relying on hosted merge and close flows.

## Fixed

- Fixed an integration behavior mismatch where branch PR guidance could imply squashing task history even though preserving history is safer for auditability.
- Fixed branch PR lifecycle ambiguity by introducing a resolver for active lifecycle context instead of scattering context derivation across commands.
- Fixed PR artifact state drift by moving artifact state into a typed representation.
- Fixed partial PR-open failure risk by making PR creation transactional across local and hosted steps.
- Fixed stale public workflow documentation that still described intermediate or legacy branch PR behavior.
- Fixed CLI resume output inconsistency in remaining direct stdio surfaces.
- Fixed release-plan coverage gaps by documenting all commits listed between `v0.3.28` and `v0.3.29`.

## Upgrade Notes

- No manual migration is required for existing `0.3.28` projects.
- Branch PR integrations now preserve task branch history by default; use `--merge-strategy squash` only when intentionally compacting branch history.
- Projects and agents that relied on squash-by-default branch PR merges should update their local guidance and prompts to match the new default.
- Public workflow documentation has been updated to describe the current branch PR lifecycle and hosted close behavior.
