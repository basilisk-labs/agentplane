# Release Notes — v0.3.2

## Summary

- Hardened the release flow after the `0.3.1` cycle so local push and publish preparation are less likely to stall on recursive hook execution or branch-name assumptions.
- Improved upgrade and doctor ergonomics for partially updated repositories by making managed-file apply the default path and by surfacing clearer recovery guidance.
- Tightened direct workflow closure so finishing a task no longer leaves task README metadata dirty by default in normal direct-mode use.
- Added framework-checkout runtime handoff: when `agentplane` from `PATH` is launched inside the framework repository, it now delegates to the repo-local binary by default and documents how to opt out.

## Added

- Added framework-checkout runtime detection and repo-local binary resolution for the CLI wrapper.
- Added end-to-end regression coverage for repo-local handoff, including nested working directories, opt-out behavior, and recursion protection.
- Added an agent-onboarding scenario check that keeps bootstrap docs, lifecycle guidance, recovery guidance, and docs navigation aligned.
- Added a dedicated legacy-upgrade recovery page and linked it from user and troubleshooting docs.
- Added explicit docs guidance that `AGENTS.md` and `.agentplane/policy/**` are framework-managed, while `.agentplane/policy/incidents.md` remains the sanctioned local override area.
- Added a canonical generated agent bootstrap document that keeps the shortest startup path synchronized across docs and CLI help.
- Added startup-surface drift checks so bootstrap docs, quickstart output, and role guidance stay aligned before release.
- Added explicit docs IA guidance for agent-first onboarding, lifecycle navigation, and recovery entrypoints.

## Improved

- Improved release apply so its internal push step bypasses recursive local hook execution.
- Improved release regression coverage by making the push-path test branch-agnostic instead of assuming `main` in temporary repositories.
- Improved the command bootstrap path by aligning `AGENTS.md`, quickstart, role help, docs navigation, and generated bootstrap guidance to one agent-first model.
- Improved user docs so the default happy path stays short while rare manual flags are isolated into explicit fallback sections.
- Improved framework-checkout UX by making `agentplane` from `PATH` behave like the current checkout instead of silently using an older global install.
- Improved upgrade messaging to print the active source, version, and execution mode before applying managed framework files.
- Improved docs navigation so the public site mirrors the task and recovery flow that the CLI expects agents to follow.
- Improved onboarding ergonomics by publishing a single copy-paste bootstrap command block instead of multiple competing entrypoints.

## Fixed

- Fixed `doctor` so historical task archive commit drift is downgraded to warnings where appropriate instead of blocking current repository health.
- Fixed `doctor` diagnostics for incomplete managed policy trees by pointing directly to the shortest recovery path.
- Fixed `upgrade` default behavior so managed framework files apply immediately unless the caller explicitly chooses `--dry-run` or `--agent`.
- Fixed direct-mode task closure so normal `finish` usage records the task metadata and closes it without leaving a dirty working tree by default.
- Fixed docs and CLI guidance drift around setup profiles, hooks defaults, upgrade behavior, task-doc planning, and framework-checkout runtime behavior.
- Fixed the release CLI regression where local `release apply --push --yes` could hang after creating the commit and tag but before finishing the push.
- Fixed the release test harness so temporary repositories no longer assume a `main` default branch during push verification.
- Fixed framework-checkout invocation so nested directories inside the repository still resolve to the repo-local binary correctly.

## Upgrade Notes

- No breaking API or workflow-mode changes are introduced in `0.3.2`.
- Framework developers using a global `agentplane` install should update the installed CLI to pick up the new repo-local handoff behavior.
- Repositories that were partially upgraded or manually patched should run `agentplane doctor` and then `agentplane upgrade --yes` or `agentplane upgrade --remote --yes` as indicated by diagnostics.
- Teams upgrading older initialized projects should expect `doctor` to flag hybrid states where `AGENTS.md` references managed policy modules that are still missing on disk.
- Direct-mode users should expect `finish` to create the close commit automatically unless they explicitly opt out.
- Release maintainers should treat `docs/releases/vX.Y.Z.md` as a required authored artifact before `agentplane release apply`.

## Verification

- Release plan generated from `v0.3.1` with `agentplane release plan --patch`.
- Repo-local checks covered release prepublish, docs-site generation/build/design checks, onboarding parity checks, wrapper regressions, and the full pre-push gate before release apply.
- npm/GitHub publish verification runs after the release commit and tag are pushed.
- The release scope includes the finished agent-first docs alignment work, repo-local wrapper handoff, direct-finish cleanup, and post-`0.3.1` release-flow hardening.
- Wrapper regressions covered repo-local handoff from `PATH`, nested framework directories, recursion guards, and explicit global opt-out.
- Recovery guidance was verified against the docs bootstrap and troubleshooting surfaces so agents can follow one consistent upgrade path.
