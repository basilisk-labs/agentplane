# Release Notes — v0.3.4

## Summary

- Made the npm-installed startup path self-contained so installed repositories no longer depend on repo-only docs artifacts during onboarding.
- Improved installed-agent ergonomics by modernizing the bundled development prompts and by making `agentplane role <ROLE>` render from the installed profile surface first.
- Repaired upgrade behavior for older initialized repositories so `agentplane upgrade --yes` restores required workflow runtime artifacts without breaking minimal or partially initialized fixtures.

## Added

- Added workflow-artifact restoration to the normal upgrade path for initialized repositories, including `.agentplane/WORKFLOW.md` and `.agentplane/workflows/last-known-good.md`.
- Added a shared workflow-artifact helper so init, upgrade, and workflow build paths derive the same runtime workflow context instead of maintaining separate mutation logic.
- Added role-help rendering that uses the installed agent profile as the primary source and appends only narrow CLI/runtime supplements.

## Improved

- Improved `AGENTS.md`, `agentplane quickstart`, and role guidance so the primary install-first startup path for npm users points to runtime commands instead of repo-only docs files.
- Improved installed development prompts to be more explicit about decision boundaries, evidence, escalation, and residual risk for real product work.
- Improved the public docs around installed agents and legacy-upgrade recovery so the expected order of operations is clearer for repositories that were initialized on older framework versions.

## Fixed

- Fixed the startup-surface bug where shipped guidance referenced `docs/user/agent-bootstrap.generated.mdx`, a docs-site artifact that is not shipped in the npm package and is not installed into user repositories.
- Fixed the upgrade gap where legacy initialized repositories could finish `agentplane upgrade --yes` without recreating required workflow runtime artifacts.
- Fixed the workflow-restore path so it only runs when the repository is initialized enough to own installed agent profiles, which prevents minimal upgrade fixtures and safety tests from failing under the broader upgrade path.

## Upgrade Notes

- No breaking workflow or configuration changes are introduced in `0.3.4`.
- For installed repositories, the canonical startup path is `AGENTS.md` -> `agentplane quickstart` -> `agentplane role <ROLE>`. The docs-site bootstrap page remains a public docs artifact, not a required local file.
- Repositories upgraded from older versions should now regain `.agentplane/WORKFLOW.md` and `.agentplane/workflows/last-known-good.md` during `agentplane upgrade --yes` when the repository is already initialized and owns the managed agent/workflow tree.
- Minimal or partially initialized fixtures continue to skip workflow-artifact restoration until the managed agent surface exists, so upgrade safety behavior remains deterministic for tests and bootstrap edge cases.
