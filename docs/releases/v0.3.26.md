# Release Notes - v0.3.26

## Summary

- Locks the `0.3.x` surface after the freeze work: CLI lifecycle, init, task close flows, config schema naming, package contents, and module topology now have an explicit freeze artifact and ADR.
- Shrinks the published `agentplane` package surface by relying on the bundled CLI runtime and a narrow package whitelist instead of shipping broad TypeScript compiler output.
- Consolidates the init and lifecycle paths so the release has one supported init orchestrator, documented close taxonomy, and release-critical coverage for the direct task lifecycle.
- Adds post-freeze hygiene guards for Knip debt, legacy test-helper suffixes, and release parity so follow-up `0.3.x` patches cannot silently widen the frozen surface.

## Added

- Added `FREEZE.v0.3.md` as the root v0.3 surface record, including command inventory, lifecycle map, config schema pointer, package boundary, and CLI checksum.
- Added ADR 0011 to define the accepted v0.3 surface freeze, frozen compatibility areas, and allowed internal refactoring boundaries before v0.4.
- Added release parity enforcement that requires `FREEZE.v0.3.md` to exist and reference the current `agentplane@0.3.x` package version.
- Added a dedicated release-critical direct lifecycle integration test that exercises `init`, `task new`, `task plan set`, `task plan approve`, `task start-ready`, `task verify-show`, `verify`, and `finish`.
- Added user and developer close taxonomy documentation that explains `finish`, close-noop, close-duplicate, hosted-close, hosted-close-pr, and PR close variants.
- Added developer module topology documentation so the remaining shared layers have explicit ownership, import direction, and scope boundaries.

## Improved

- Improved package publishing by pruning compiler-emitted JavaScript before the `agentplane` bundle is produced, making `dist/cli.js` the runtime source of truth.
- Improved npm install behavior by removing the package `prepare` hook and relying on `prepack` and `prepublishOnly` for publication builds.
- Improved the published `agentplane` file list so `bin/*.d.ts`, source maps, and broad internal build output stay out of the tarball.
- Improved init consistency by replacing the parallel v1/v2 orchestrator split with a single orchestrator path.
- Improved non-init prompt handling by moving remaining prompt consumers onto the shared Clack-backed adapter names.
- Improved CLI test ownership by moving repo-private run-cli helper modules into `@agentplane/testkit` instead of leaving excluded helper files under `agentplane/src`.
- Improved release bootstrap checks so the bootstrap freshness scripts can read source TypeScript modules under Bun while the published runtime remains bundled.
- Improved local CI routing after moving the ANSI helper into `src/shared`, keeping focused test selection aligned with the new module path.
- Improved config schema discoverability by renaming the core Zod implementation to `schema.impl.ts` and removing the thin schema shim.
- Improved close and PR precheck reuse by centralizing shared close-family validation in a command-shared helper.
- Improved task lifecycle maintainability by splitting the `verify-record` and `hosted-close.command` hotspots below the release hotspot threshold.
- Improved Knip debt control by ratcheting the unused-code baseline to the current post-cleanup counts.
- Improved repository hygiene by keeping the active `agentplane-recipes` submodule metadata and documenting that the empty-submodule premise is stale for this checkout.
- Improved local type clarity by recording that `bin/*.d.ts` remains a repository-local TypeScript declaration surface while the package whitelist keeps it unpublished.
- Improved test naming hygiene by adding a hard ESLint guard against future `*.test-helpers.ts` files.

## Fixed

- Fixed release parity drift by making `FREEZE.v0.3.md` fail when it does not match the current `0.3.x` package version.
- Fixed the package artifact mismatch where the CLI bundle and broad compiler output could both be present in the published package.
- Fixed init UI drift by removing v2-named files and orphaned init preview exports after the unified init path landed.
- Fixed stale init export noise by pruning unused UI preview helpers that no longer had call sites.
- Fixed close-family duplication by sharing precheck behavior between PR close and hosted-close PR flows without changing CLI result semantics.
- Fixed task lifecycle documentation gaps by adding close taxonomy coverage to the user lifecycle documentation.
- Fixed legacy helper drift by removing `agentplane` tsconfig exclusions that existed only for test helper files.
- Fixed shared-layer ambiguity by documenting the shared directory topology before reducing the number of `shared` directories.
- Fixed the CLI shared helper split by moving the ANSI helper out of `cli/shared` and into the cross-cutting shared layer.
- Fixed release-critical coverage gaps by registering the new lifecycle integration test in the release-critical suite.

## Upgrade Notes

- Surface locked for 0.3.x: public CLI behavior, `.agentplane/` task artifacts, config field compatibility, and package boundary changes should remain patch-compatible until v0.4.
- No manual migration is required for normal projects that already run on `0.3.25`.
- Framework maintainers should run `bun run framework:dev:bootstrap` after pulling this release because release-critical runtime and release metadata changed during the freeze work.
- Projects using direct lifecycle automation should treat `task start-ready`, `task verify-show`, `verify`, and `finish` as the canonical direct lifecycle sequence.
- Consumers should continue importing public core subpaths internally; root core import compatibility remains for external consumers under ADR 0010.
