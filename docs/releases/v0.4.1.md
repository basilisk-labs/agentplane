# Release Notes - v0.4.1

## Summary

- Ships the first patch after `v0.4.0`, focused on making recipes visible and installable through the current CLI release path.
- Rotates the production recipes index trust root to the `2026-05` Ed25519 key and documents the custody and rotation rules for future catalog signing keys.
- Publishes the `code-map@0.1.0` recipe catalog path and keeps the default remote catalog compatible with the updated AgentPlane trust model.
- Consolidates the public recipes documentation so top-level `Recipes` describes signed installable packages while copy-paste agent setup pages live under `Workflow guides`.
- Adds release distribution groundwork for public listings and Homebrew/Scoop package handoff.
- Tightens the release and test surface around prompt modules, recipes, branch PR closure, and release-specific CI so future patch releases have clearer evidence.

## Added

- Added `code-map@0.1.0` as the single production recipe in the connected recipes catalog.
- Added a release-ready recipes signing root for key id `2026-05`.
- Added a lightweight code-map recipe behavior that changes agent work by asking it to map relevant code before editing.
- Added prompt selector merge diagnostics so prompt-module resolution is easier to inspect.
- Added a modular check runner and route registry so release-critical checks are easier to split and maintain.
- Added docs information-architecture drift guards for current documentation and website sidebar coverage.
- Added a dedicated `Code Map` recipe page and a top-level `Workflow guides` section for agent/editor setup pages.
- Added release distribution scripts and docs for rendering Homebrew formula assets and publishing package manager handoff metadata.
- Added Scoop bucket manifest rendering and publish-workflow artifact wiring for Windows package manager handoff.
- Added public listing documentation for external AI coding agent directories.
- Added GPT-5.5 prompt contract diagnostics and shared prompt fragments.
- Added homepage hero motion and release-oriented website refinements after the 0.4 launch page.

## Improved

- Improved recipe catalog compatibility by aligning the submodule, default catalog signature, CLI trust root, and runtime version checks.
- Improved release confidence by splitting large guard, hook, PR, release, and task query test suites into narrower release-critical units.
- Improved release readiness by refreshing the Knip unused-code baseline after reviewing prompt-module public API drift.
- Improved branch PR close evidence by closing follow-up tasks through hosted PRs and refreshing task artifacts after verification commits.
- Improved prompt-module maintenance by extracting the mutation engine, normalizing prompt compiler context, and adding schema migration structure.
- Improved website presentation by simplifying the launch homepage, restoring menu rounding, and adding relevant background motion.
- Improved package and website navigation so recipe links point at the implemented catalog shape instead of workflow-copy pages.
- Improved public release readiness by recording external listing PRs and package manager publication tasks before tagging `0.4.1`.
- Improved external directory coverage by recording submissions to `sorrycc/awesome-code-agents` and `filipecalegario/awesome-vibe-coding`.
- Improved package README metadata so npm package surfaces match the current repository state.

## Fixed

- Fixed the default recipes catalog path so a freshly installed CLI can trust and list the current production index once `0.4.1` is published.
- Fixed the recipes catalog signature mismatch caused by the earlier development key id.
- Fixed the oversized test guard baseline and related lint drift discovered during the refactor wave.
- Fixed release apply and branch PR test monoliths by splitting slow or mixed suites into focused coverage files.
- Fixed generated prompt and init asset parity gaps found during the final refactor-wave verification.
- Fixed documentation drift around agent-first IA, release workflow evidence, recipes safety, and key rotation.
- Fixed the misleading top-level recipes section that grouped Claude Code, Codex, Cursor, Aider, GitHub Actions, and `branch_pr` guides as recipes.
- Fixed Docusaurus route links for the new recipes and workflow guide index pages.
- Fixed a formatting drift in the active incident policy mirror that blocked the release heavy gate.

## Upgrade Notes

- No manual migration is required for projects already on `0.4.0`.
- Install `agentplane@0.4.1` to pick up the `2026-05` recipes trust root used by the default production catalog.
- Older `0.4.0` installations may still reject the current default recipes index unless they use an override trust root or upgrade to this patch.
- Recipe signing private keys must stay out of the repository and out of project `.env` files; the production online storage location is the GitHub Actions secret documented in the developer recipes docs.
