# Release Notes - v0.3.28

## Summary

- Reduces the public and internal TypeScript surface by pruning obsolete barrel exports across incident handling, Redmine sync/runtime paths, runner adapters, release preflight, bootstrap guidance, run-cli command loading, and recipes commands.
- Lowers the Knip baseline by removing internal-only exports that were no longer part of an active contract.
- Stabilizes the CLI cold-start guard used by pre-push and local CI by increasing warmups and retry attempts without relaxing latency thresholds.
- Carries forward the v0.3.27 publication closeout so the patch history remains complete from the previous release tag.

## Added

- Added release documentation for the v0.3.28 patch so the generated release plan has human-readable coverage for each included commit.
- Added explicit release coverage for the v0.3.27 closeout commit to keep the release delta auditable even when the previous publication close landed after the tag boundary.

## Improved

- Improved pre-push stability for the CLI cold-start baseline by running the default guard with two warmups and five attempts while keeping the existing median latency ceilings unchanged.
- Improved the incident command surface by pruning shared exports that were only used internally, reducing accidental API reach and Knip noise.
- Improved Redmine task backend maintainability by removing unused sync re-exports and runtime barrel exports from the public import graph.
- Improved runner adapter boundaries by pruning adapter re-exports that were not part of the active consumer contract.
- Improved release preflight maintainability by removing the obsolete preflight barrel and routing call sites to direct modules.
- Improved bootstrap guide internals by pruning exports that were not needed outside their implementation path.
- Improved run-cli command loading by removing the remaining core command barrel and keeping command catalog imports more explicit.
- Improved recipes command boundaries by removing the recipes command barrel and keeping implementation imports closer to their owning modules.
- Improved the Knip baseline by removing internal-only exports, reducing the unused-export budget for future refactoring work.
- Improved v0.3 baseline cleanup documentation and alignment so the refactoring plan, docs, and enforcement surfaces describe the current repository state more accurately.

## Fixed

- Fixed a release-process flake where the CLI cold-start guard could fail pre-push on transient local timing spikes even though a standalone retry passed with the same thresholds.
- Fixed stale public-surface pressure from internal-only re-exports that made unused-export reports harder to interpret.
- Fixed documentation drift for the scripts inventory by regenerating `scripts/README.md` after the cold-start guard command changed.

## Upgrade Notes

- No manual migration is required for existing `0.3.27` projects.
- The CLI cold-start guard remains strict on the same latency thresholds; the patch only changes the default warmup and retry budget used by the check command.
- Consumers should continue importing from documented subpaths instead of relying on internal barrels or incidental root re-exports.
