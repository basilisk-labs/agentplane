# Release Notes - v0.3.25

## Summary

- Fixed the interactive `agentplane init` crash that occurred when the cached recipe selection prompt received an undefined validation value during the TTY flow.
- Added focused regression coverage for the cached recipe selection path so the failing init dialog is now exercised both at the prompt-step layer and in the interactive init route.
- Carried the hotfix branch through the same traceable task lifecycle used by the rest of the `0.3` line, including formatting and lint recovery needed to satisfy the protected publication path.
- Preserves the previously published `v0.3.24` baseline while shipping the new init hotfix as the next patch release.

## Added

- Added a prompt-step regression test that validates cached recipe selection remains stable when the prompt validator receives `undefined` before submit.
- Added an interactive init regression test that covers the legacy cached-recipe path and asserts the TTY dialog no longer crashes before writing `.agentplane/config.json`.
- Added explicit release-task traceability for publishing this hotfix branch so the release range records the publication, formatting recovery, and lint recovery as first-class task outcomes.

## Improved

- Improved cached recipe selection resilience by treating non-string validation input as an empty answer and falling back to the preset recipe selection instead of throwing.
- Improved release traceability by carrying the prior `v0.3.24` publication closeout evidence forward into the `v0.3.25` range instead of leaving the previous release-task closure outside the next patch baseline.
- Improved hotfix publication reliability by bringing the new init regression tests into repository formatting and lint compliance before the protected pre-push route ran.

## Fixed

- Fixed the `TypeError: Cannot read properties of undefined (reading 'trim')` crash in the `Materialize cached recipes` prompt during interactive `agentplane init`.
- Fixed the cached recipe prompt parser so validation-time `undefined` values no longer abort the init flow when legacy cached manifests are present.
- Fixed formatting drift in the new init regression tests so protected pre-push checks accept the hotfix branch.
- Fixed lint drift in the same regression mocks by removing unnecessary `async` wrappers that violated the repository lint gate.
- Fixed release-range continuity by including the completed `v0.3.24` publication closeout commits in the `v0.3.25` change window rather than treating them as invisible background history.

## Upgrade Notes

- No manual migration is required for normal projects.
- Projects that hit the interactive cached-recipe init crash can update directly to `v0.3.25`.
- Framework maintainers should keep using `bun run framework:dev:bootstrap` before publication when watched runtime files changed during the hotfix.
