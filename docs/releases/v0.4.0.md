# Release Notes - v0.4.0

## Summary

- Introduces modular prompt assembly as the new Agentplane prompt architecture: fragments, modules, a compiler, registries, graph diagnostics, and generated prompt assets now share one model.
- Extends recipes toward module-level prompt changes so future recipes can target named behavior instead of patching whole prompt files.
- Makes init, runner, policy gateway, and generated agent assets compile from the modular prompt graph.
- Closes the 0.3 line with a retrospective blog post and prepares 0.4 as the first release where prompt modules are a product boundary.

## Added

- Added prompt fragment parser contracts for named, addressable prompt material.
- Added a prompt module compiler that resolves modular prompt graphs into deterministic compiled outputs.
- Added a framework prompt module registry so canonical prompt modules can be discovered and loaded consistently.
- Added init-time compilation for gateway policy modules from prompt modules.
- Added init-time compilation for agent profiles from prompt modules.
- Added runner prompt module bridging so runtime prompt assembly can use the same graph concepts.
- Added recipe manifest fields for prompt module declarations and mutation sets.
- Added recipe prompt mutation application against the compiled graph.
- Added recipe fragment selectors for module-level recipe targeting.
- Added prompt graph diagnostics and drift checks.
- Added modular prompt migration documentation and task graph guidance.
- Added release and hosted-close evidence diagnostics for the new release path.
- Added a website retrospective that marks `v0.3.29` as the final published 0.3 release and explains why 0.4 starts at prompt modules.

## Improved

- Improved prompt assembly by replacing large opaque prompt files with named modules that can be selected, validated, and compiled.
- Improved recipe extensibility by preparing recipes to operate on prompt modules and fragment selectors.
- Improved generated agent and policy consistency by syncing mirrors from canonical fragmented prompt assets.
- Improved release readiness by reconciling generated prompt outputs before publish.
- Improved branch PR closure evidence by recovering hosted closure deltas before the 0.4 release boundary.
- Improved verification metadata around prompt assembly tasks, release hygiene tasks, and hosted close tasks.
- Improved documentation of prompt fragment naming, mutation scope, and migration sequencing.
- Improved runner prompt behavior by adopting prompt modules without changing the public task workflow.
- Improved init behavior by compiling gateway policy and agent profiles from the same prompt module source model.
- Improved doctor feedback by exposing prompt graph diagnostics and drift checks.
- Improved task traceability by keeping task artifacts refreshed after each prompt migration step.
- Improved release story clarity by publishing the `Agentplane 0.3: the road to 0.4` blog post before this release.
- Improved release gate discipline by treating prompt module hotspot debt and generated mirror drift as publication blockers rather than post-release cleanup.

## Fixed

- Fixed fragmented prompt mutability enforcement so recipe and module mutations respect replacement boundaries.
- Fixed generated agent and policy mirror drift after the prompt fragmentation migration.
- Fixed release formatting drift discovered during patch-readiness checks.
- Fixed missing hosted closure deltas before preparing the 0.4 release candidate.
- Fixed release evidence diagnostics that could otherwise leave task verification state ambiguous.
- Fixed stale task artifacts after prompt compiler, registry, init, runner, recipe, and documentation changes.
- Fixed migration documentation gaps by publishing the prompt fragment task graph and contract docs before release.
- Fixed release framing by moving the modular prompt milestone from a planned `0.3.30` patch to `0.4.0`.

## Upgrade Notes

- No manual migration is required for existing `0.3.29` projects.
- Projects that regenerate Agentplane assets through init or upgrade paths will receive prompt assets compiled from the modular prompt graph.
- Recipe authors can start designing around prompt modules, selectors, and mutation sets, but should continue to follow the documented recipe manifest contract.
- The `0.3` maintenance branch points at `v0.3.29`; the modular prompt architecture starts in the `0.4` line.
