# Release Notes - v0.3.17

## Summary

- Completed the audit-driven patch roadmap by turning the v0.3.16 audit, delta, refactoring plan, and removal inventory into tracked implementation tasks.
- Removed obsolete legacy compatibility paths so new projects and upgrades converge on the current task, policy, workflow, and documentation contracts.
- Reduced command and schema hotspots across hooks, work-start, hosted close, Redmine runtime, recipes, and task artifacts without changing the supported CLI surface.
- Strengthened release readiness with dependency drift guards, stricter lint gates, expanded unused-code checks, and release notes that make the migration impact explicit.

## Added

- Added a workspace dependency drift guard to release parity so package and lockfile changes are caught before publish.
- Added production console usage inventory enforcement so new runtime logging must go through the approved logger path.
- Added a no-console guard that keeps production CLI output intentional while preserving explicit command output helpers.
- Added a dependency-cruiser baseline gate so architecture violations cannot grow silently.
- Added a Knip baseline guard that fails when unused files, exports, types, enum members, or namespace members increase beyond the accepted debt.
- Added explicit signing algorithm validation for recipe indexes, making `ed25519` the only accepted algorithm in recipe metadata.
- Added shared recipe manifest and type modules so recipe validation, loading, and command code can depend on a smaller public surface.
- Added shared Zod validation-error formatting for core config and task artifact schema failures.
- Added nested task artifact validation coverage so comments and other structured paths report actionable locations.
- Added focused hosted-close PR command modules for precheck, execution, postcheck, reporting, and shared types.
- Added split hook runner modules for install, shared constants, dispatch, commit-message checks, pre-commit checks, pre-push checks, and post-merge cleanup.
- Added split work-start modules for direct-mode locks, base-branch freshness checks, repo-local hook shim materialization, and worktree bootstrap materialization.
- Added release-plan evidence for `v0.3.17` from `v0.3.16`, including the generated change list and patch target.
- Added release notes for the roadmap task graph so the patch has a human-readable migration and verification record before publication.

- Improved config deprecation reporting by routing warnings through the project logger instead of direct console writes.
- Improved task artifact schema maintainability by splitting the large schema file into shared, task, handoff, findings, verification, and PR metadata domains.
- Improved Redmine backend runtime maintainability by separating context, state, operations, methods, and mapping responsibilities.
- Improved hosted-close PR readability by keeping the command entrypoint small and delegating the workflow stages to cohesive modules.
- Improved hooks command maintainability by turning `hooks/index.ts` into a facade over specialized modules.
- Improved work-start maintainability by keeping `cmdWorkStart` as the coordinator and moving support behavior into dedicated modules.
- Improved large CLI test maintainability by splitting recipe tests into install-project, validation-list, and remote-usage scenario families.
- Improved query-run-execute test maintainability by splitting Codex, custom runner, control, and history scenarios into separate files.
- Improved process supervision lint compliance by enabling full `no-misused-promises` checking and removing the async event-listener pattern.
- Improved release parity by validating workspace dependency drift as part of the release-adjacent gate.
- Improved generated schema discipline by removing the obsolete schema diff helper and relying on the current schema synchronization path.
- Improved Vitest dependency stability by pinning the Vitest toolchain used by the workspace.
- Improved testkit dependency hygiene by correcting drift that could otherwise make local and package-level tests disagree.
- Improved release readiness documentation by recording the patch roadmap backlog and closeout metadata as traceable task artifacts.
- Improved the CLI test migration surface by carrying forward shared fixture and helper lessons from previous migration incidents.
- Improved project skill discovery by emitting metadata-only project skill blocks during runner base prompt assembly.
- Improved repo-local recovery workflows by adding a closure recovery skill for finish, hosted-close, close-tail, PR mutation, and task-state recovery.
- Improved testkit migration guidance by documenting the canonical `@agentplane/testkit` naming and migration path.
- Improved release and packaging operator guidance with a repo-local skill that records the expected package and release workflow.
- Improved PR artifact freshness for the runner skill discovery work so review state reflects the latest task evidence.
- Improved release readiness by refreshing the framework dev runtime after watched runtime and command files changed.
- Improved CLI help stability by keeping public command specs unchanged while reorganizing their implementation internals.

## Fixed

- Fixed legacy task-index v1 acceptance by removing the old compatibility reader and requiring the current task index contract.
- Fixed legacy root `WORKFLOW.md` compatibility by removing the fallback path and relying on the active policy gateway.
- Fixed old upgrade source redirect behavior by removing the stale redirect and associated doctor recovery path.
- Fixed task README compatibility drift by making task doc v3 the only accepted task README version.
- Fixed temporary legacy warning instrumentation drift by removing the planned warning phase in favor of the approved legacy removal route.
- Fixed direct console warning drift by routing config warnings through logger-backed output.
- Fixed hosted-close command concentration that made precheck, execution, postcheck, and reporting harder to verify independently.
- Fixed Redmine backend runtime concentration that mixed mapping, state, context, and operations in one larger implementation surface.
- Fixed schema error inconsistency by replacing custom config and task-artifact issue text with one formatter.
- Fixed hook command concentration that made hook install, uninstall, run, and per-hook behavior harder to audit.
- Fixed work-start command concentration that mixed direct-mode, branch-pr, worktree, runtime bootstrap, and hook-shim behavior.
- Fixed release readiness blind spots by requiring release notes coverage before the patch can move to publication.

## Upgrade Notes

- Upgrades from `v0.3.16` to `v0.3.17` intentionally drop several legacy compatibility paths.
- Projects must use the current task index format; legacy task-index v1 data is no longer accepted.
- Projects must use the current policy gateway and workflow documentation path; root `WORKFLOW.md` compatibility has been removed.
- Projects must use task README doc v3; older task README document contracts are no longer accepted.
- Upgrade and doctor recovery logic no longer follows the old upgrade source redirect path.
- Deprecated CLI flags now fail explicitly instead of being accepted through compatibility shims.
- Recipe index metadata must use the supported `ed25519` signing algorithm.
- The release keeps these removals in a patch because the upgrade operator is expected to handle old-project migration explicitly.
