# Release Notes - v0.3.23

## Summary

- Hardened `agentplane init` so stale cached recipe manifests no longer abort interactive setup before the project is created.
- Moved hook-install conflict detection earlier in both init orchestration paths so hook collisions fail before Agentplane writes project files.
- Aligned framework bootstrap hook shims with the current installed-runner contract so maintainer rebuilds no longer reintroduce stale shim warnings.
- Carries forward the merged v0.4 prompt-assembly groundwork and related task-graph/docs artifacts into the next patch baseline.

## Added

- Added early hook-install conflict probing so `init --hooks yes` can surface unmanaged `.git/hooks/*` and shim collisions before writing `.agentplane/config.json`.
- Added regression coverage for malformed cached recipe manifests that are missing both `prompts` and `scenarios`, proving init now prunes those entries instead of failing with `E_IO`.
- Added regression coverage for legacy cached recipe scenarios that still omit newer scenario fields and confirmed the migration path stays intact.
- Added focused v2 init conflict coverage so the interactive Clack flow now previews hook conflicts before apply instead of discovering them only after writes begin.
- Added bootstrap regression assertions that require the framework-maintainer shim template to embed the installed runner path and the opt-in `AGENTPLANE_HOOK_ALLOW_NPX` fallback.
- Added task-graph and release-plan traceability entries for the prompt-assembly groundwork that landed in `main` before this patch cut.
- Added runtime-aware subprocess resolution for release and hook flows so internal `bun` and `node` calls prefer the user's working toolchain instead of drifting onto broken fallback runtimes.

## Improved

- Improved init cache loading so only the init-specific cached-recipe listing path becomes lenient; the public `recipes` commands stay strict and continue rejecting invalid cache state.
- Improved installed-recipe migration by normalizing valid entries and dropping malformed ones in a stable write-back pass, leaving the cache in a cleaner post-init state.
- Improved non-interactive init behavior by treating hook conflicts as ordinary init conflicts, so users get the standard conflict message and existing `--force` or `--backup` recovery path.
- Improved the default TTY init route so hook conflicts show up in the same conflict-resolution UX as file collisions instead of appearing as a late command failure.
- Improved framework bootstrap reconciliation so a maintainer `bun run framework:dev:bootstrap` no longer rewrites `.agentplane/bin/agentplane` back to an older shim contract.
- Improved the `knip` release guardrail by refreshing the baseline to the reviewed post-merge debt level now present on `main`, preventing false release blocks from stale thresholds.

## Fixed

- Fixed the init crash reported as `Invalid field manifest: expected prompts or scenarios` when the cached global recipe manifest had degraded into metadata-only entries.
- Fixed the partial-init failure mode where hooks could fail only after earlier init steps had already written project files.
- Fixed stale bootstrap-generated shims that were missing the installed-runner fallback branch expected by `agentplane doctor`.
- Fixed the bootstrap shim quoting contract so doctor can recognize the embedded installed runner path after framework rebuilds.
- Fixed the release baseline to include the integrated v0.4 prompt-module contract commits, associated doc updates, and policy mirror sync without losing them between patch releases.

## Upgrade Notes

- No manual migration is required for normal projects.
- Repositories with stale cached recipe entries can simply re-run `agentplane init`; invalid cache records are now pruned during init.
- Repositories with custom unmanaged git hooks still need an explicit overwrite or backup decision during init.
