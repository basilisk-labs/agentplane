# Release Notes - v0.3.20

## Summary

- Released a patch hotfix for interactive `agentplane init` so cached recipes from older Agentplane versions no longer abort setup when scenario metadata omits modern fields such as `use_when`.
- Restored the original Agentplane init banner in the Clack-backed dialog.
- Added full-dialog and critical process smoke coverage so the default init path is exercised before future patch releases.

## Added

- Added a critical CLI smoke test that runs `agentplane init --yes` with a legacy cached recipe manifest and verifies setup completes without schema errors.
- Added full TTY dialog coverage for the default init choices, including the cached legacy recipe path and final apply confirmation.

## Improved

- Improved schema v1 cached recipe compatibility by deriving safe defaults for modern scenario metadata fields that older cache entries did not write.
- Improved init UI regression coverage so the original banner remains visible during interactive setup.

## Fixed

- Fixed `agentplane init` failing with `Invalid field manifest.scenarios[0].use_when: expected string[]` when `AGENTPLANE_HOME` contains cached recipes written by earlier Agentplane versions.
- Fixed the incomplete `v0.3.19` compatibility shim, which handled missing scenario `name` but still rejected other legacy scenario fields.
- Fixed the init banner regression by restoring the historical Agentplane logo block.

## Upgrade Notes

- Projects upgrading from `v0.3.19` do not need manual migration.
- Users do not need to delete existing `AGENTPLANE_HOME/recipes.json` cache files before running `agentplane init`.
- The change is limited to init recipe-cache compatibility, init UI, and regression coverage.

## Verification

- `bun run test:project -- recipes packages/recipes/src/index.test.ts` passed for legacy and strict recipe manifest validation.
- `bun run test:project -- agentplane packages/agentplane/src/cli/init-recipes-cache.test.ts packages/agentplane/src/cli/run-cli/commands/init/ui-v2.test.ts` passed for init cache and logo coverage.
- `bun run test:project -- cli-core packages/agentplane/src/cli/run-cli.core.init.v2.test.ts` passed for the full default TTY init dialog.
- `bun run test:project -- critical packages/agentplane/src/cli/run-cli.critical.exit-codes.test.ts` passed for real-process init smoke coverage.
- `bun run --filter=@agentplaneorg/recipes typecheck && bun run --filter=agentplane typecheck` passed.
- `bun run release:parity` passed before release application.
