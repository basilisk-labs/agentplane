# Release Notes — v0.3.0

## Summary

- Delivered the policy-gateway refactor and modular routing model for agent workflows.
- Introduced stronger release-note quality controls so generated notes now target full change coverage.
- Improved installation and startup behavior for light profile users and linked global CLI usage.
- Hardened CI and publishing paths for idempotent release reruns and clearer release prerequisites.

## Added

- Added policy gateway/module contract hardening so gateway templates, routing rules, and checks stay aligned.
- Added a rebuilt AGENTS gateway layout with explicit routing into dedicated policy modules.
- Added routed policy checker enforcement tied to task-doc quality gates.
- Added explicit conditional-loading requirements for gateway policy modules.
- Added single-source incident logging semantics with explicit routing to one incidents log.
- Added init/upgrade support for bundled policy templates installed directly from framework assets.

## Improved

- Improved policy routing clarity by consolidating governance behavior around one canonical incident log.
- Improved prompt and workflow maintainability by modularizing AGENTS gateway routing paths.
- Improved release-note authoring instructions to require concrete, human-readable bullets per listed change.
- Improved release-note validation behavior to enforce detailed coverage instead of minimal generic notes.
- Improved task lifecycle closure consistency by finalizing DONE-status artifact handling.
- Improved developer ergonomics for linked global CLI usage by relaxing stale-dist blocking outside the framework checkout.
- Improved setup defaults for light profile by disabling hooks and simplifying initial welcome output.
- Improved documentation consistency with refreshed generated CLI reference output.

## Fixed

- Fixed commit-msg hook flow so stale-dist checks no longer block commit-msg hook execution.
- Fixed publish workflow behavior to allow reruns when the release tag already exists on origin.
- Fixed release idempotency by allowing publish continuation when release tags were already created.
- Fixed CI ordering by building package manifests before CLI docs freshness checks.
- Fixed policy-loading ambiguity by enforcing condition-driven module loading with stricter routing checks.

## Upgrade Notes

- No breaking API changes were introduced in this release.
- Repositories using policy modules should run routing checks after upgrade to validate local customizations.

## Verification

- Release plan generated from `v0.2.26` with `agentplane release plan --minor --yes`.
- Release notes coverage expanded to include all listed changes in the plan artifacts.
- Core release gates are expected to run via `bun run release:prepublish` before publish.
