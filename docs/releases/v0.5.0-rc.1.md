# Release Notes — v0.5.0-rc.1

## Summary

- Adds the first executable blueprint contract layer without making code PR flow the default for all task types.
- Connects recipe blueprint extensions to the resolver as advisory, validated route inputs.
- Exposes blueprint route decisions through CLI explain output, task verification guidance, and ACR summaries.
- Keeps runner execution out of scope for this release candidate.

## Added

- Added built-in blueprint routes for `analysis.light`, `content.light`, `docs.change`,
  `code.direct`, `code.branch_pr`, `release.strict`, and `ops.approval`.
- Added resolver support for task metadata, workflow mode, mutation scope, risk flags, explicit
  blueprint requests, and recipe hints.
- Added `agentplane blueprint list` and `agentplane blueprint explain` with human and JSON output.
- Added recipe `blueprint_extensions` bridge support for context hints, evidence requirements, check
  suggestions, output schemas, artifact templates, risk hints, and compatible blueprint preferences.
- Added blueprint expected evidence to `agentplane task verify-show`.
- Added compact blueprint summary fields to ACR records under `extensions["agentplane.blueprint"]`.

## Improved

- Analysis and content tasks now have lightweight route contracts that do not include worktrees, PRs,
  hosted checks, merge gates, or CI by default.
- Recipe preferences are accepted only when compatible with the inferred task kind and workflow
  mode; stronger risk routes still win.
- Verification guidance now shows the resolved evidence contract before verification is recorded.

## Fixed

- Prevented recipe hints from acting as lifecycle overrides or protected-node replacements.
- Prevented incompatible recipe preferences from forcing analysis/content work into code PR routes.

## Upgrade Notes

- No migration is required for existing projects.
- The new blueprint surfaces are explain and evidence surfaces only; they do not execute blueprint
  nodes or replace `direct` / `branch_pr`.

## Verification

- Focused blueprint resolver and validation tests cover built-in route selection, recipe extension
  acceptance/rejection, preferred blueprint compatibility, and risk-priority behavior.
- CLI smoke checks cover `blueprint list`, synthetic `blueprint explain`, task-based
  `blueprint explain --json`, and `task verify-show` blueprint evidence output.
- Full release verification must still run the repository fast local gate before tagging rc1.
