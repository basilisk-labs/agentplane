# Release Notes - v0.3.21

## Summary

- Released a patch hotfix for `agentplane init` so legacy cached recipe scenarios that omit `file` metadata no longer abort interactive setup.
- Added cache migration during the init cached-recipes read path, so old `AGENTPLANE_HOME/recipes.json` entries are normalized and written back after AgentPlane sees them.
- Closed the previous `v0.3.20` release task trail and carried its init compatibility evidence forward into this patch.

## Added

- Added regression coverage for legacy schema v1 recipe scenarios with no `file` field, including manifest normalization and cache writeback.
- Added full interactive init coverage that verifies a legacy cached recipe manifest is migrated while the default dialog completes.

## Improved

- Improved schema v1 recipe compatibility by deriving a deterministic fallback scenario file path from the normalized scenario id.
- Improved the init cache path so successful normalization persists to `recipes.json`, preventing the same old cache entry from failing future runs.

## Fixed

- Fixed `agentplane init` failing with `Invalid field manifest.scenarios[0].file: expected string` when cached recipes were written by older AgentPlane versions.
- Fixed the remaining compatibility gap after `v0.3.20`, where legacy scenario defaults covered modern descriptive fields but still required modern file metadata.

## Upgrade Notes

- Users do not need to delete or manually edit existing `AGENTPLANE_HOME/recipes.json` files.
- The migration is limited to cached recipe manifests read during init; strict schema v2 recipe manifests still require explicit scenario `file` metadata.

- `bun run test:project -- cli-core packages/agentplane/src/cli/run-cli.core.init.v2.test.ts` passed for the full interactive init dialog with legacy cache migration.
