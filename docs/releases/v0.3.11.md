# Release Notes — v0.3.11

## Summary

- `v0.3.11` is a workflow-hardening patch focused on branch_pr lifecycle reliability, protected-main closeout recovery, and PR artifact consistency.
- The patch tightens task-state, incident-promotion, and verification ergonomics so repeated closure and reconciliation work leaves less manual cleanup behind.
- GitHub auth propagation, worktree bootstrap, stale-dist handling, cleanup flows, and remote-check waiting all received follow-up fixes during this release window.
- There is no intended repository-data migration in `v0.3.11`; the main operator-visible change is a more stable and auditable workflow surface.

## Added

- Added concrete Verify Steps seeding for verify-required task scaffolds so fresh tasks can be approved without manual README repair.
- Added pre-merge validation for committed branch-backed PR artifacts so integrate fails before base-branch mutation when required metadata is missing.
- Added clearer guidance when `pr open` is attempted from an unpushed task branch so operators can correct the branch state before remote PR creation.
- Added protected-main close-wave reconciliation coverage so superseded task PRs, closure branches, and merge evidence can be converged deterministically.
- Added explicit support for deleting remote task branches from cleanup flows once the relevant merged state is confirmed.
- Added stronger progress-aware remote-check waiting so long GitHub check runs surface live status instead of looking stalled.
- Added broader branch_pr artifact preservation and recovery paths for duplicate closures, already-DONE tasks, and stale base snapshots.
- Added tighter incident-promotion reporting so local-only findings, promoted findings, and no-op outcomes are distinguished explicitly.

## Improved

- Improved fresh branch_pr worktrees so repo-local runtime/bootstrap state is seeded more consistently before task execution starts.
- Improved closeout reliability by making finish and integrate more tolerant of stale-dist, partial retries, and duplicate closure scenarios.
- Improved PR-artifact freshness so normalize, pr check, pr update, and post-commit refresh paths stay aligned with the real task branch head.
- Improved GitHub auth propagation so repo dotenv state no longer overrides a valid `gh` login during hosted-merge and PR flows.
- Improved branch and worktree cleanup so merged stale task branches can be pruned without extra manual origin fetch and resolution steps.
- Improved incident advice ergonomics so operators see exact next actions, incident IDs, and promotion scope more clearly during verify/finish.
- Improved auditability of release-window closeouts by preserving more deterministic task artifacts, closure comments, and reconciliation evidence in git history.
- Improved release and recovery diagnostics by fixing local release E2E auth handling and keeping the release-ready/publish trail easier to inspect.

## Fixed

- Fixed the late-integrate failure mode where base-branch mutation could start before the task branch had committed the required PR artifacts.
- Fixed finish-side mutation ordering so task docs are no longer rewritten before DONE validation has actually passed.
- Fixed stale verify and closeout guidance gaps so no-op and warning paths now point to the correct recovery action.
- Fixed multiple PR publication and synchronization edge cases around unpushed branches, stale snapshots, notes storage, and post-commit refresh.
- Fixed several protected-main closure tails by reconciling missing follow-up PRs, stale closure artifacts, and superseded branch state.
- Fixed branch_pr auth/env leakage that previously forced manual token workarounds even when a valid `gh` session already existed.
- Fixed worktree/bootstrap rough edges so legacy hook installs and missing runtime state degrade cleanly instead of failing obscurely.
- Fixed release-surface regressions that could still break local recovery or exact-ref validation before the final publish gate.
