# Release Notes — v0.3.10

## Summary

- `v0.3.10` ships the full framework control-plane foundation that was planned in the F-001 through F-010 roadmap.
- It adds a reusable incident-memory layer: finished tasks can now promote structured external incident advice into `.agentplane/policy/incidents.md`, and new tasks can receive targeted incident guidance before execution.
- This patch also rolls in the post-`v0.3.9` workflow, PR-artifact, and release-documentation fixes that landed after the previous tag.
- There is no intended task-data migration in `v0.3.10`; the main change is stronger runtime structure, better operator guidance, and a more auditable release/workflow surface.

## Added

- Added `ResolvedHarnessContract` as the explicit framework harness layer so repo policy, workflow rules, approvals, protected paths, and backend restrictions now resolve through one runtime contract.
- Added `AgentplaneExecutionContext` as the canonical runtime context so command, runner, and framework flows stop rebuilding ad hoc execution state.
- Added the capability registry/runtime so commands, tools, agents, and related surfaces resolve through one formal capability model instead of scattered implicit checks.
- Added the framework behavior precedence core so harness, extension, user, and builtin behavior now resolve through one traceable precedence path.
- Added the expanded policy taxonomy layer so risky and mutating actions share one clearer classification surface.
- Added the approval runtime so approval semantics for risky operations resolve through one first-class framework gateway.
- Added executable execution profiles so budgets, stop conditions, handoff conditions, timeout rules, and trace policy are now runtime data instead of partial convention.
- Added task-intake contracts for clarification, draft graph, and materialization planning so `task create`-style flows have stable framework types to build on.
- Added explain hooks that expose resolved harness, policy, capability, and intake inputs as explicit framework output.
- Added protocol/result foundation types so framework explain data and related runner artifacts now have additive machine-readable envelopes.
- Added synced framework documentation across the README, architecture docs, harness guidance, and the framework refactor program so the shipped control-plane foundation is documented as implemented code instead of future intent.
- Added the incidents runtime and CLI surface so `agentplane incidents collect` and `agentplane incidents advise` can promote and query structured incident knowledge directly.
- Added finish-time incident promotion so structured external `incident-candidate` findings can be appended to `.agentplane/policy/incidents.md` without manual file surgery.
- Added task-start incident advice lookup so `agentplane task start-ready` can surface relevant historical incident guidance by scope and tags before new work begins.
- Added machine-readable incident registry fields such as `tags`, `match`, `advice`, `source_task`, and `fixability`, and synced that contract into the shipped policy templates.
- Added explicit framework-roadmap task provenance in the repository so the control-plane rollout itself is traceable through seeded and integrated task records.

## Fixed

- Fixed the framework’s lack of one explicit control-plane baseline by merging the harness, context, capability, precedence, approval, execution-profile, intake, explain, and protocol tasks into one shipped runtime.
- Fixed the previous “incident log is manual memory only” gap by turning `incidents.md` into a structured registry that can both accumulate operational advice and drive targeted lookup.
- Fixed the mismatch between live repo policy and shipped init/upgrade templates by syncing the updated incident and workflow guidance into `packages/agentplane/assets/**`.
- Fixed the stale release-gate assumption that the framework refactor was still only planned by updating the public docs to describe the now-implemented runtime surfaces.
- Fixed the ambiguity around whether incident knowledge should be bulk-loaded on startup by documenting and shipping targeted lookup/promotion paths instead.
- Fixed the PR/workflow surface drift that remained after `v0.3.9` by carrying the workflow-mode sync and GitHub PR workflow work into this patch line.
- Fixed the missing PR audit trail on several post-`v0.3.9` workflow tasks by carrying their artifact refresh commits into this release.
- Fixed the post-`v0.3.9` release documentation gap by integrating the `v0.3.8` / `v0.3.9` release-blog correction work.
- Fixed the traceability of that release-blog correction by preserving its dedicated integration record in the release history.
- Fixed the broken `v0.3.8` release line in repository history by carrying forward the already-published replacement patch release `v0.3.9` evidence into the current release window.
- Fixed the gap between completed framework work and the recorded release ledger by including the roadmap seed, rollout, verification artifacts, and integration trail in the `v0.3.10` release plan.

## Upgrade Notes

- Upgrade to `v0.3.10` if you want the framework control-plane foundation, incidents registry automation, and the post-`v0.3.9` workflow hardening in one patch release.
- There is no intended repository-data migration in this patch; the main operational change is that `agentplane task start-ready` can now surface incident advice and `agentplane finish` can promote structured external incident candidates.
- Repositories that customize shipped policy templates should review the updated incident-policy contract and the new `agentplane incidents collect` / `agentplane incidents advise` commands after upgrading.
