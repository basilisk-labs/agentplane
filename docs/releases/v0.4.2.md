# Release Notes - v0.4.2

## Summary

- Ships the next patch after `v0.4.1`, focused on release reliability, recipe catalog trust, and homepage recipe visibility.
- Adds the `2026-06` recipes catalog public key so newly signed remote indexes can be trusted by the CLI.
- Improves external distribution publication metadata so package-manager and action recovery paths report clearer source references.
- Confirms the public homepage recipes section renders the remote catalog with install commands for each listed recipe.
- Closes the Node 24 standalone release check tail after targeted standalone distribution tests passed on the current main line.

## Added

- Added the `2026-06` Ed25519 recipes index public key to the trusted recipes keyring.
- Added richer release distribution metadata for Homebrew, Scoop, setup-agentplane, and external publication handoff scripts.
- Added release candidate coverage for the latest recipe catalog homepage and install-command behavior.

## Improved

- Improved recipe listing confidence by verifying that the public site loads the live remote catalog and renders recipe cards.
- Improved standalone release validation by recording the Node 24 check-mode archive optimization as verified on main.
- Improved commit and PR message format handling through the merged `93GVV1` branch-pr title and close-tail fixes.
- Improved release distribution recovery by keeping exact release SHA and module metadata visible in publishing artifacts.
- Improved release readiness by carrying forward standalone installer, package-manager, GHCR, and setup-action distribution work from `v0.4.1`.

## Fixed

- Fixed the stale task tail for the Node 24 standalone release script work by merging the deterministic closeout PR.
- Fixed the default recipes index trust gap for indexes signed with the new `2026-06` key.
- Fixed release metadata gaps that made external distribution publishing less transparent during recovery.
- Fixed homepage recipe install command formatting through the already-merged recipes catalog homepage work.

## Upgrade Notes

- No manual project migration is required for projects already on `0.4.1`.
- Upgrade to `agentplane@0.4.2` to trust recipes indexes signed with key id `2026-06`.
- Existing recipe caches can continue to use already cached packages; refresh the remote index to see newly signed catalog updates.
- Release operators should continue using exact SHA based hosted publication for patch releases.
