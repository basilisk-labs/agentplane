# Release Notes - v0.3.24

## Summary

- Hardened the `0.3` foundation line by removing stale `v0.4` backlog pressure, tightening guardrail inventories, and keeping the active docs aligned with the real stabilization scope.
- Reduced several high-noise hotspots in guard, release, and task-doc migration code so the current maintenance line is easier to verify and extend without semantic drift.
- Stabilized local CI cold-start checks with clearer retry and ordering behavior, reducing false negatives in the pre-push fast path.
- Carries forward the completed `v0.3.23` release task into the next patch baseline together with the post-release hardening that landed immediately after publication.

## Added

- Added a dedicated README-migration helper module for `task migrate-doc`, separating task-doc rendering and normalization from workspace orchestration.
- Added shared aggregate test-target inventories for significant coverage and workflow-harness coverage so both guardrails read one canonical source.
- Added explicit documentation that the current active line is `0.3.23` foundation stabilization and that deeper prompt-assembly work is deferred.
- Added clearer documentation labeling for the prompt-assembly system page so it reads as a deferred design note instead of active runtime behavior.
- Added retry-aware parsing for failed cold-start measurement payloads so the baseline checker can explain malformed attempts instead of collapsing them into opaque failures.
- Added deterministic cold-start gate ordering inside local CI so the fast path stays stable when benchmark steps and downstream checks run together.

## Improved

- Improved `task migrate-doc` maintainability by shrinking the main runtime file from 461 lines to 191 while keeping the command entrypoint stable.
- Improved foundation guardrail maintainability by removing duplicated significant-suite target lists from the coverage scripts.
- Improved workflow-harness coverage maintenance by moving its watched targets into the same shared inventory module used by other routing checks.
- Improved developer docs so `framework-refactor-program`, architecture guidance, and docs IA now describe the current foundation-first release line instead of stale `v0.4` assumptions.
- Improved recipe docs so active `0.3.23` runtime behavior is separated from deferred prompt-assembly and compiler ideas.
- Improved the docs index so deferred prompt-assembly material is clearly marked as non-active design guidance.
- Improved release preflight readability by splitting `release apply` preflight work into focused helper modules instead of one larger hotspot.
- Improved guard commit maintainability by extracting focused commit orchestration helpers from the previous hotspot implementation.
- Improved cold-start baseline behavior by adding warmup and retry budgeting to the CLI benchmark guard instead of relying on single-shot measurements.
- Improved task-board focus by retiring the non-active prompt-assembly carryover from the live `0.3` foundation backlog before continuing stabilization work.

## Fixed

- Fixed duplicate aggregate guardrail ownership so the significant-coverage and workflow-harness checks no longer drift independently when targets change.
- Fixed stale foundation docs that still presented prompt-assembly and recipe expansion work as active near-term implementation guidance.
- Fixed stale `release apply` preflight lint drift by removing an obsolete import from the release path.
- Fixed the previous `release apply` hotspot by decomposing its preflight checks into narrower modules with clearer ownership boundaries.
- Fixed the previous `guard commit` hotspot by splitting refresh and staging responsibilities into dedicated helpers.
- Fixed the cold-start retry guard so failed measurement payloads are parsed and reported instead of producing weaker diagnostics.
- Fixed local CI cold-start sequencing so the fast path keeps the intended check order even as the CLI baseline step grows more defensive.
- Fixed the `bench:cli:cold:check` path so warmup and retry behavior reduce noisy variance in pre-push and local-CI runs.
- Fixed task-doc migration hotspot pressure by moving README normalization logic out of the workspace-level command file.
- Fixed release-baseline continuity by carrying the completed `v0.3.23` release closure forward into the `v0.3.24` release range.

## Upgrade Notes

- No manual migration is required for normal projects.
- Teams using the developer docs should treat the prompt-assembly pages as deferred design notes unless a later release explicitly promotes them into active runtime guidance.
- Repositories depending on local framework-maintainer rebuilds should keep using the canonical rebuild path before direct task-lifecycle commits when runtime files change.
