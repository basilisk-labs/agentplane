# Release Notes - v0.3.19

## Summary

- Released a patch hotfix for interactive `agentplane init` so stale cached recipes from earlier versions no longer abort setup before project files are written.
- Restored the ASCII Agentplane logo in the Clack-backed init flow, matching the expected visual entry point for the interactive setup.
- Added focused regression coverage for legacy recipe scenario descriptors that provide `summary` but omit `name`.
- Carried forward the completed `v0.3.18` release close record so the release history remains traceable from the previous published patch.

## Added

- Added recipe manifest compatibility coverage for cached recipe descriptors with legacy scenario entries that do not include a `name` field.
- Added init recipe-cache regression coverage proving that `listCachedRecipes()` accepts older cached manifests instead of failing during init startup.
- Added init UI coverage for the restored ASCII logo so future Clack UI changes keep the expected banner visible.
- Added release notes for the `v0.3.19` hotfix path from `v0.3.18`.

## Improved

- Improved cached recipe manifest normalization by deriving a missing scenario `name` from the scenario `summary` when reading legacy data.
- Improved interactive init presentation by showing the ASCII Agentplane banner immediately after the Clack intro.
- Improved release traceability by including the previous patch-release close commit in the generated release history.
- Improved test clarity around the init logo assertion so formatting checks pass without weakening the regression.

## Fixed

- Fixed `agentplane init` failing with `Invalid field manifest.scenarios[0].name: expected string` when `AGENTPLANE_HOME` contains cached recipes written by older Agentplane versions.
- Fixed interactive init aborting before recipe selection because cached recipe validation was stricter than the legacy cache shape.
- Fixed the missing ASCII logo regression in the default interactive init experience.
- Fixed the logo regression test formatting so the release can pass the standard lint and formatting gates.

## Upgrade Notes

- Projects upgrading from `v0.3.18` do not need manual migration.
- Existing cached recipe files are read compatibly; users do not need to delete `recipes.json` to run `agentplane init`.
- The change is limited to init UI and recipe manifest compatibility.

- `bun run test:project -- recipes packages/recipes/src/index.test.ts` passed for recipe manifest normalization.
- `bun run test:project -- agentplane packages/agentplane/src/cli/init-recipes-cache.test.ts packages/agentplane/src/cli/run-cli/commands/init/ui-v2.test.ts packages/agentplane/src/cli/run-cli/commands/init/prompts-v2.test.ts` passed for init cache, logo, and prompt coverage.
- `bun run --filter=@agentplaneorg/recipes typecheck` passed.
- `bun run --filter=agentplane typecheck` passed.
- `bun run framework:dev:bootstrap` passed before the release branch was prepared.
- A smoke run with a temporary legacy `AGENTPLANE_HOME/recipes.json` completed `agentplane init --yes` successfully.
