# Release Notes - v0.3.22

## Summary

- Hardened installed AgentPlane usage for clean user projects, with self-contained hook fallback behavior and clearer doctor diagnostics when managed hook shims are stale.
- Strengthened workflow guardrails around task verification, lifecycle errors, commit allowlists, and generated command guidance.
- Added release-facing test coverage for installed-runtime smoke paths, verify flag behavior, test routing, oversized test ratchets, and hook readiness checks.
- Carries forward the v0.3.21 release closure metadata into the next patch baseline.

## Added

- Added an installed-runtime smoke gate that exercises the packaged CLI path instead of only the repository-local development entrypoint.
- Added doctor diagnostics for stale managed hook shims so users can see when a clean project is blocked by old hook bootstrap code.
- Added a test inventory source of truth, test routing guard, oversized test ratchet, and focused coverage for verify flag combinations.

## Improved

- Improved `agentplane quickstart` and generated guidance so installed-user workflows remain self-contained and avoid repo-only assumptions.
- Improved lifecycle and verification error guidance so failed commands point to the next concrete AgentPlane action.
- Improved commit allowlist behavior so approved task artifacts can be staged correctly even when the index already contains other staged changes.
- Improved generated task guidance to prefer canonical `bun run test:project ...` commands instead of broad or ambiguous test invocations.

## Fixed

- Fixed installed pre-push hook fallback behavior for clean projects that do not contain repository-local hook scripts.
- Fixed branch-pr worktree hook shim handling so generated worktrees get the expected managed hook behavior.
- Fixed a release-readiness hotspot by moving doctor hook readiness checks into a focused module.
- Closed the previous v0.3.21 release task and included that closure in the release-plan coverage for this patch.

## Upgrade Notes

- No manual migration is required.
- Projects with stale AgentPlane-managed hooks should run the doctor guidance and reinstall hooks when prompted.
