# Release Notes — v0.4.4

## Summary

- Ships the next patch release after the v0.4.3 baseline with all completed launch, documentation, diagnostics, and workflow tasks included.
- Switches the binary distribution path for this release to Bun single-file executable assets across GitHub Release installers and external package-manager renderers.
- Keeps npm package publication aligned for `agentplane`, `@agentplaneorg/core`, and `@agentplaneorg/recipes` at the same patch version.
- Records release evidence so package-manager and setup-action channels can be checked against the same release manifest.

## Added

- Added Bun executable release assets as the release-owned binary channel for macOS arm64, macOS x64, Linux arm64, Linux x64, and Windows x64.
- Added release workflow upload coverage for `bun-assets.json` and all `agentplane-bun-*` archives.
- Added archive smoke support to the Bun compiled CLI smoke script so published Bun archives can be validated directly.

## Improved

- Changed generated install scripts to default to the Bun executable channel instead of standalone bundled-Node archives.
- Changed Homebrew formula rendering to consume Bun executable assets for macOS platforms.
- Changed Scoop manifest rendering to consume the Windows Bun executable archive and expose `bin\agentplane.exe`.
- Changed setup-agentplane rendering to install Bun executable archives with checksum validation.
- Updated external channel metadata to mark Bun single-file executables as the default install strategy.

## Fixed

- Fixed the release-distribution mismatch where Bun assets could appear in `release-distribution.json` and `SHA256SUMS` without being uploaded as GitHub Release assets.
- Fixed package-manager renderer assumptions that only standalone platform assets had public release URLs.

## Upgrade Notes

- No configuration migration is required for npm users.
- Shell and PowerShell installers now default to Bun executable archives; set `AGENTPLANE_INSTALL_CHANNEL=bun` explicitly only when pinning the channel in automation.
- Homebrew, Scoop, and setup-agentplane outputs for this release are expected to reference Bun executable archives.

## Verification

- `bun run release:distribution:check` passed after switching release metadata to Bun assets.
- `bun run release:homebrew:check`, `bun run release:scoop:check`, and `bun run release:setup-action:check` passed against Bun asset metadata.
- Publish workflow must confirm that uploaded GitHub Release assets include all `agentplane-bun-v0.4.4-*` archives and `bun-assets.json`.
