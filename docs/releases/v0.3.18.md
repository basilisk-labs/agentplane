## Upgrade Notes

- This patch is expected to be compatible with projects already on `v0.3.17`.
- Projects using the current CLI, task document v3, and current policy gateway do not need manual migration for this release.
- Internal imports should continue moving away from broad `@agentplaneorg/core` root imports and toward public subpaths.
- Repo-local tests and helpers should consume `@agentplane/testkit` through public package exports rather than deep source or dist paths.
- The init interactive experience now uses the v2 Clack-backed route by default for TTY sessions; automation and non-TTY paths keep the established behavior.
- The prompt assembly documentation describes planned v0.4 architecture and should not be treated as a completed runtime migration.
- The obsolete `v0.3.16` release branch remains historical and is not part of the `v0.3.18` upgrade path.
