# Release Notes — v0.3.9

## Summary

- `v0.3.9` is primarily a release-hardening patch that clears the runway for the `0.4` and `0.5` release lines.
- It also replaces the broken `v0.3.8` npm artifact, where `agentplane@0.3.8` leaked a `workspace:` dependency on `@agentplane/recipes` and made `npm install agentplane` fail before the CLI could start.
- `v0.3.9` removes that publish-time dependency edge, hardens release validation around installability, and tightens local hook/runtime fallback behavior used by release and upgrade flows.

## Added

- Added a release parity guard that fails when the publishable `packages/agentplane/package.json` manifest contains `workspace:` dependencies.
- Added regression coverage for the exact leaked-manifest shape that escaped in `v0.3.8`.

## Improved

- Improved publish safety by aligning the shipped CLI package with the runtime code that already existed inside `packages/agentplane`.
- Improved release confidence by making installability part of the publishable manifest contract instead of an implicit assumption.
- Improved hook-runner fallback order so managed hook shims prefer an explicit runner or repo-local CLI before falling back to `agentplane` in `PATH` or `npx`.
- Improved the release/upgrade smoke path so fixture repositories and managed hooks do not depend on whatever npm version of `agentplane` happens to be globally visible at test time.

## Fixed

- Fixed the `npm install` failure introduced in `v0.3.8`, where npm aborted with `EUNSUPPORTEDPROTOCOL` on `workspace:packages/recipes`.
- Fixed the packaging graph so the published CLI no longer depends on the private workspace package `@agentplane/recipes`.
- Fixed the missing release gate that allowed a non-installable npm manifest to pass through the `0.3.8` publish flow.

## Upgrade Notes

- If you attempted to install `agentplane@0.3.8` and hit `EUNSUPPORTEDPROTOCOL`, install `agentplane@0.3.9` or newer instead.
- There is no intended workflow or task-data migration in `v0.3.9`; this patch is about restoring a valid install path, hardening release infrastructure, and reducing risk before the larger `0.4` and `0.5` lines.
- Operators already on `v0.3.7` should skip `v0.3.8` and move directly to `v0.3.9`.

## Verification

- Inspected the published `agentplane@0.3.8` npm manifest and confirmed that it shipped `@agentplane/recipes=workspace:packages/recipes`.
- Built and packed the fixed CLI locally, inspected the packed manifest, and confirmed that the unsupported workspace dependency was gone.
- Installed the fixed tarball with plain `npm install` in a clean temporary directory and verified that the CLI help output starts normally.
- Ran targeted hook, upgrade, and release-smoke tests so the repaired hook-runner fallback and installability guard remained green.
- Ran `bun run release:prepublish` on the exact release tree before publish.
