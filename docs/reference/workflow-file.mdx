# Workflow file reference

Agentplane keeps workflow and verification policy in repo-local files so coding-agent work can be
reviewed after the session ends. `.agentplane/WORKFLOW.md` is the workflow/config file; CLI surfaces
such as `agentplane config show` read it back rather than defining another source of truth.

Open the detailed configuration reference:

- [Configuration and workflow files](../user/configuration)

The main local files are:

```text
AGENTS.md
.agentplane/WORKFLOW.md
agentplane.yaml
```
