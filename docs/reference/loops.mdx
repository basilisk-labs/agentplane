# Loops Reference

Loops are controlled feedback cycles for AgentPlane tasks. They sit between
blueprint route selection and runner/agent execution.

## Discovery

```bash
agentplane loop list
agentplane loop list --json
agentplane loop show tdd.fix
agentplane loop explain tdd.fix
```

Use `--project` to include JSON loop specifications from `.agentplane/loops`.
Project-local loops are validated before they are listed.

## Planning

```bash
agentplane loop plan <task-id>
agentplane loop plan --kind code --tag bug --tag test --verify-steps-present --approved-plan
agentplane loop plan <task-id> --json
```

Planning is read-only. It prints the selected loop, score, reasons, and rejected
candidates. If no candidate reaches the threshold, the command exits non-zero
and asks for an explicit route decision.

Candidate selection applies hard `requires` gates before ranking. Synthetic
planning must state facts such as `--approved-plan`, `--clean-worktree`,
`--hosted-pr`, and `--ci-failure`; missing required facts make a candidate
ineligible rather than merely lowering its score.

## Execution modes

```bash
agentplane loop run <task-id> --loop tdd.fix --dry-run
agentplane loop step <task-id> --loop tdd.fix --dry-run
agentplane loop step <task-id> --loop tdd.fix --execute-agent-step
agentplane loop run <task-id> --loop tdd.fix --execute
agentplane loop run <task-id> --loop tdd.fix --resume <run-id>
```

- `--dry-run` prepares evidence without invoking an agent.
- `--execute-agent-step` is the legacy one-step runner handoff and stops before
  diff capture, checks, and evaluation.
- `--execute` runs the supported deterministic loop to a terminal decision.
- `--resume` reloads the durable cursor and continues without replaying
  completed steps.

The executable runtime currently supports `tdd.fix`. Other built-in and
project-local loops remain discoverable and validatable but fail closed if
passed to `--execute`.

Dry-run output creates:

```text
.agentplane/tasks/<task-id>/runs/<run-id>/loop-run.json
.agentplane/tasks/<task-id>/runs/<run-id>/events.jsonl
.agentplane/tasks/<task-id>/runs/<run-id>/state.json
.agentplane/tasks/<task-id>/runs/<run-id>/iterations/001/decision.json
```

Executable runs use the same layout, adding per-step input/output artifacts and
an atomically updated cursor in `state.json`. Every external step is followed by
a checkpoint before the next transition is evaluated.

## Budgets and context

`tdd.fix` enforces iteration, wall-time, changed-file, diff-line, agent-run,
token, and no-progress limits. Runner trace token usage is aggregated into the
LoopRun state when the adapter exposes it.

The agent step receives the exact rendered prompt and its digest. Its runner
bundle keeps only `Summary`, `Scope`, `Plan`, and `Verify Steps`; previous
iteration feedback is carried as a bounded delta rather than replaying raw task
comments and events.

## Validation

```bash
agentplane loop validate .agentplane/loops/custom.json
agentplane loop validate --project
agentplane loop validate --project --json
```

Loop JSON may use camelCase fields or the snake_case aliases used in docs:
`schema_version`, `applies_to`, and `stop_conditions`.

## Current non-goals

- no automatic loop activation or promotion;
- no policy weakening from loop specs;
- no arbitrary transition expressions outside the deterministic condition
  registry;
- no executable runtime for loops other than `tdd.fix` until they have their own
  step executors and regression coverage.
